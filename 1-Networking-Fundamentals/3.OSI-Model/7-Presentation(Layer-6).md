# OSI Layer 6 - Presentation Layer

## Overview
The **Presentation Layer** is responsible for making sure data is in a **standard format** that both sending and receiving devices can understand.

It acts like a **translator** between applications and the network.

---

## Main Role
- Converts data into a readable format
- Ensures different systems understand the same data
- Handles data formatting and structure

---

## Why It is Needed

Different applications may store or send data in different ways, but the receiver must still understand it correctly.

Example:
- One email client sends a message
- Another email client receives it
- Both must display the same content correctly

---

## Key Functions

### 1. Data Translation
- Converts data between formats
- Ensures compatibility between systems

### 2. Data Formatting
- Structures data for proper display and use

### 3. Data Encryption
- Secures data before transmission
- Example: **HTTPS encryption**

---

## Security Role

The Presentation Layer is responsible for:
- Encrypting data before sending
- Decrypting data after receiving

This ensures secure communication over networks.

---

## Key Takeaway

The **Presentation Layer (Layer 6)** standardizes, translates, and encrypts data so that different systems can understand and securely exchange information.
