# UDP/IP (User Datagram Protocol)

## What is UDP?

**UDP (User Datagram Protocol)** is a simple, fast, and connectionless protocol used to send data between devices.

Unlike TCP, it does not create a stable connection before sending data.

---

## Main Idea

- No connection setup (no handshake)
- No confirmation of delivery
- No tracking of packets
- Just sends data and moves on

👉 “Send it and hope it arrives”

---

## UDP vs TCP (Simple Difference)

| TCP | UDP |
|-----|-----|
| Connection-based | Connectionless |
| Reliable | Unreliable |
| Slower | Faster |
| Error checking | No guarantee |
| Ordered data | No order guarantee |

---

## Advantages of UDP

- Very fast
- Low overhead (less processing)
- Good for real-time applications
- Flexible for developers

---

## Disadvantages of UDP

- No guarantee data will arrive
- No error correction
- Packets may be lost
- Unstable connection affects quality badly

---

## When UDP is Used

UDP is used when speed matters more than perfection:

- Video streaming
- Online gaming
- Voice calls
- Live broadcasts

---

## UDP Packet Structure (Simple Headers)

UDP packets are simpler than TCP packets.

### Time to Live (TTL)
- Sets expiry time for packet
- Prevents network congestion

---

### Source Address
- IP address of sender

---

### Destination Address
- IP address of receiver

---

### Source Port
- Random port used by sender

---

### Destination Port
- Port of the receiving service (e.g., 80 for web servers)

---

### Data
- Actual information being sent

---

## Key Difference: No Connection Setup

UDP does NOT use:
- Three-way handshake
- Acknowledgement system
- Synchronisation between devices

👉 Data is sent instantly without setup.

---

## Key Takeaway

UDP is a **fast but unreliable protocol** that sends data without establishing a connection, making it ideal for real-time applications where speed is more important than accuracy.
