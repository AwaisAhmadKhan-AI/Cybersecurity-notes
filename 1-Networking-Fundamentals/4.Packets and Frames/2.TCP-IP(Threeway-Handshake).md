# TCP/IP & Three-Way Handshake

## What is TCP/IP?

**TCP/IP (Transmission Control Protocol / Internet Protocol)** is a set of rules used for sending data across networks.

It is a simplified model compared to OSI and contains **4 layers**:

- Application
- Transport
- Internet
- Network Interface

---

## TCP/IP vs OSI (Simple Idea)

- OSI Model → 7 layers (detailed explanation)
- TCP/IP Model → 4 layers (real-world simplified version)

Both models use **encapsulation**, meaning data gets extra information at each layer.

---

## What is TCP?

**TCP (Transmission Control Protocol)** is a connection-based protocol.

It ensures:
- Data is delivered correctly
- Data is complete
- Data is in order

👉 TCP is used when reliability is important.

---

## TCP Key Features

- Connection-based (must establish connection first)
- Reliable delivery
- Error checking
- Ordered data transmission
- Slower than UDP due to extra checks

---

## Advantages of TCP

- Very reliable
- Ensures data integrity
- Maintains correct order of packets
- Prevents data loss

---

## Disadvantages of TCP

- Slower than UDP
- Uses more system resources
- Requires stable connection
- Can delay transmission if packets are lost

---

## Important TCP Headers

### Source Port
- Port used by sender

### Destination Port
- Port used by receiving application (e.g., 80 for web servers)

### Source IP
- IP address of sender

### Destination IP
- IP address of receiver

### Sequence Number
- Number assigned to track order of packets

### Acknowledgement Number
- Confirms received data and tells next expected packet

### Checksum
- Detects corrupted data

### Data
- Actual content being sent

### Flags
- Control how packets behave (important for connection setup/closing)

---

# Three-Way Handshake (TCP Connection Setup)

## What is it?

The **Three-Way Handshake** is the process used by TCP to establish a secure connection between two devices before sending data.

---

## Steps of Three-Way Handshake

### 1. SYN (Synchronize)
- Client sends a request to start connection
- Includes initial sequence number

---

### 2. SYN-ACK
- Server responds
- Acknowledges client request
- Sends its own sequence number

---

### 3. ACK (Acknowledgement)
- Client confirms server response
- Connection is now established

---

## After Connection is Established

Once handshake is complete:
- DATA packets are exchanged
- Sequence numbers ensure correct order

---

## Simple Sequence Idea

- Client starts with number (e.g., 0)
- Each packet increases by +1
- Both devices agree on starting numbers during handshake

---

# Closing a TCP Connection

## FIN (Finish)

- Used to close a connection properly

### Process:
1. Device sends FIN
2. Other device acknowledges
3. Other device sends FIN back
4. First device acknowledges

---

## RST (Reset)

- Abruptly stops connection
- Used when something goes wrong
- No proper closure process

---

# Key Takeaway

TCP/IP is a real-world networking model.

- **TCP ensures reliable communication**
- **Three-Way Handshake establishes a connection before data transfer**
- **FIN closes connections properly, while RST forcefully ends them**
