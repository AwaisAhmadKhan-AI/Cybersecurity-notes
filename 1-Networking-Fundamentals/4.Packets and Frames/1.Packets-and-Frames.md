# Packets and Frames

## Overview
Packets and frames are **small pieces of data** that together form a complete message sent across a network.

They are used to make data transfer efficient and reliable.

---

## Difference Between Packets and Frames

### Packet (Layer 3 - Network Layer)
- Contains data + IP information
- Used for routing between networks
- Works with **IP addresses**

### Frame (Layer 2 - Data Link Layer)
- Encapsulates the packet
- Adds **MAC address information**
- Used for delivery inside a local network

---

## Simple Analogy (Post Office Example)

Think of sending a letter:

- **Packet = Letter content**
- **Frame = Envelope**

Process:
1. Letter (packet) is placed inside an envelope (frame)
2. Envelope is sent to destination
3. Receiver opens envelope
4. Letter is read and processed

---

## Encapsulation Process

- Data starts at higher layers
- Each layer adds extra information
- Frame is the final wrapped form for transmission
- Receiver removes layers step by step

---

## Why Packets Are Used

- Break large data into smaller pieces
- Prevent network congestion
- Improve speed and efficiency
- Allow easier error handling

Example:
- Images, videos, and files are sent in small packets
- They are rebuilt at the destination

---

## Packet Structure (IP Example)

Packets contain important headers that help delivery:

### 1. Time to Live (TTL)
- Prevents packets from looping forever
- Packet expires after a certain time

### 2. Checksum
- Checks data integrity
- Detects if data is corrupted during transmission

### 3. Source Address
- IP address of sender

### 4. Destination Address
- IP address of receiver

---

## Key Idea

- Packets = Network Layer (IP-based data units)
- Frames = Data Link Layer (MAC-based delivery units)

---

## Key Takeaway

Packets and frames are small data units used to send information across networks. Packets handle **IP-level communication**, while frames handle **local delivery using MAC addresses**.
