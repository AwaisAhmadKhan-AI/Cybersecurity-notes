# Ports (Port 101)

## What are Ports?

Ports are **numerical endpoints** used for communication between devices and applications.

Think of a **harbour**:
- Ships = data
- Harbour = device
- Ports = docking points

Only compatible ships can dock at specific ports — same rule applies in networking.

---

## Main Idea

- A device can have many applications running
- Ports help separate and organize traffic
- Each service uses a specific port number

👉 Ports ensure data goes to the correct application.

---

## Port Range

- Ports range from **0 to 65535**
- Each port is assigned to a specific service or application

---

## Why Ports are Needed

Without ports:
- All data would mix together
- Applications wouldn’t know which data belongs to them
- Communication would break down

Ports solve this by:
- Assigning data to correct services
- Keeping traffic organized

---

## Well-Known (Common) Ports (0–1024)

These ports are standardized and widely used:

### FTP (File Transfer Protocol) - Port 21
- Used for transferring files between client and server

---

### SSH (Secure Shell) - Port 22
- Used for secure remote login (command-line access)

---

### HTTP (Web Traffic) - Port 80
- Used for loading websites (non-secure)

---

### HTTPS (Secure Web Traffic) - Port 443
- Encrypted version of HTTP

---

### SMB (File Sharing) - Port 445
- Used for file and printer sharing in networks

---

### RDP (Remote Desktop Protocol) - Port 3389
- Used for remote graphical access to a computer

---

## Key Concept: Standard vs Custom Ports

- Applications usually follow standard ports (like 80 for HTTP)
- But they can be changed if needed

Example:
- Web server running on port **8080** instead of 80

To access it:
```
http://example.com:8080
```

👉 The colon (`:`) tells the browser which port to use.

---

## Key Takeaway

Ports are **numbered communication channels** that direct data to the correct application on a device, ensuring multiple services can run without conflict.
