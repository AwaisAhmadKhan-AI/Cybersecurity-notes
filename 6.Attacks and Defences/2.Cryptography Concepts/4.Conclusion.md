# Core Cryptography Concepts

## Plaintext

Readable, understandable data before encryption.

Example:

```text
Hello World
```

---

## Ciphertext

Scrambled, unreadable data after encryption.

Example:

```text
Xjhwjy Rjxxflj
```

---

## Key

A secret value used to:

* Encrypt data
* Decrypt data

Without the correct key, encrypted information should remain unreadable.

---

## Algorithm

The mathematical process used to:

* Transform plaintext into ciphertext
* Reverse ciphertext back into plaintext

Algorithms are generally public knowledge.

Security depends on protecting the key, not hiding the algorithm.

---

# Types of Encryption

# 1. Symmetric Encryption

Symmetric encryption uses:

> One shared key for both encryption and decryption.

---

## How It Works

* Alice and Bob both possess the same secret key
* Alice encrypts data using the key
* Bob decrypts data using the same key

---

## Advantages

* Very fast
* Efficient for large amounts of data
* Low computational overhead

---

## Disadvantages

* Difficult to securely share the key
* Vulnerable during key exchange

---

## Example

### Caesar Cipher

A simple historical encryption method where letters are shifted by a fixed number.

Example:

```text
HELLO → KHOOR
```

---

# 2. Asymmetric Encryption

Asymmetric encryption uses:

> Two mathematically linked keys.

---

## Public Key

* Shared openly
* Used for encryption

## Private Key

* Kept secret
* Used for decryption

---

## Advantages

* Solves the key distribution problem
* Public key can safely travel over the internet

---

## Disadvantages

* Slower than symmetric encryption
* Computationally expensive

---

# The Key Distribution Problem

Symmetric encryption requires both parties to already share a secret key.

Problem:

* How do you securely exchange the key in the first place?

Asymmetric encryption solves this issue by allowing:

* Public keys to be shared openly
* Private keys to remain secret

---

# Hybrid Encryption

Modern systems combine both encryption methods.

## Process

### Step 1 — Asymmetric Encryption

Used to:

* Securely exchange a symmetric key

### Step 2 — Symmetric Encryption

Used for:

* Fast communication
* Encrypting bulk data

This approach is called:

> Hybrid Encryption

---

# HTTPS and Secure Browsing

HTTPS uses both encryption methods together.

When visiting a secure website:

1. The browser receives the website's public key
2. A secure symmetric session key is established
3. Future communication uses fast symmetric encryption

This is what protects:

* Passwords
* Banking details
* Private messages
* Login sessions

---

# Certificates and Trust

Websites use digital certificates to prove identity.

Certificates contain:

* Public key
* Domain ownership information
* Certificate Authority (CA) signature

Browsers verify certificates automatically before trusting websites.
