# Hiding Information - Symmetric Encryption

## Overview

When two people want to share a secret over an insecure channel, they need a way to stop others from reading the message.

This is where **encryption** comes in.

In this lesson, we focus on **symmetric encryption**, one of the most important ideas in cryptography. Symmetric encryption allows two people to protect information by using **the same secret key** to both lock and unlock the message.

To understand it, we first need a few important terms.

---

# Core Terms

Before learning symmetric encryption, understand these four basic terms.

## Plaintext

**Plaintext** is the original readable message before it is encrypted.

### Examples

```text
HELLO
Patient name: Alice Smith
Transfer $500 to account 123
```

This is the information in its normal readable form.

---

## Ciphertext

**Ciphertext** is the scrambled, unreadable version of the message after encryption.

### Example

If the plaintext is:

```text
HELLO
```

it might become:

```text
KHOOR
```

The ciphertext should look meaningless to anyone who does not have the key.

---

## Key

A **key** is the secret value used to control how encryption and decryption work.

Think of the key as the secret password or secret number that tells the algorithm how to scramble and unscramble the message.

Without the correct key, the ciphertext should remain unreadable.

---

## Algorithm

An **algorithm** is the public set of steps used to encrypt or decrypt the data.

Important point:

> The **algorithm is not secret**.
> The **key is secret**.

This is a core idea in cryptography.

---

# Basic Encryption and Decryption Process

Cryptography usually follows this pattern:

## Encryption

```text
Plaintext + Encryption Algorithm + Key → Ciphertext
```

## Decryption

```text
Ciphertext + Decryption Algorithm + Key → Plaintext
```

So the algorithm defines **how** the scrambling works, while the key controls **which exact transformation** happens.

---

# Lockbox Analogy

A simple way to understand symmetric encryption is to imagine a **lockbox**.

Suppose Alice wants to send Bob a secret letter through the postal system, but anyone could intercept the package.

### Alice does this:

1. She writes the secret letter.
2. She places it inside a lockbox.
3. She locks the box using a key.
4. She sends the locked box through the mail.

### Bob does this:

1. He receives the locked box.
2. He uses **the same key** to unlock it.
3. He reads the letter inside.

---

## Mapping the Analogy to Cryptography

| Lockbox Example       | Cryptography Meaning |
| --------------------- | -------------------- |
| Letter inside the box | Plaintext            |
| Locked box            | Ciphertext           |
| Lock mechanism        | Algorithm            |
| Physical key          | Secret key           |

### Important Idea

The lock mechanism itself is not secret. Anyone can know how the lock works.

Security comes from keeping the **key** secret.

That is exactly how modern cryptography works:

* Algorithms are usually public.
* Keys must remain private.

---

# Plaintext vs Ciphertext Example

Suppose Alice wants to send:

```text
HELLO
```

This is the **plaintext**.

After encryption, it becomes:

```text
KHOOR
```

This is the **ciphertext**.

Bob receives the ciphertext and uses the correct key to turn it back into:

```text
HELLO
```

---

# The Caesar Cipher

To understand symmetric encryption more clearly, we use a very simple historical encryption method called the **Caesar cipher**.

> ⚠️ The Caesar cipher is **not secure** and is **not used in real systems**.
> It is only used for learning because it is simple to understand.

---

# How the Caesar Cipher Works

The Caesar cipher shifts each letter in the alphabet by a fixed number.

That fixed number is the **key**.

---

## Example Key = 3

If the key is `3`, then every letter moves **3 positions forward** in the alphabet:

| Plain Letter | Cipher Letter |
| ------------ | ------------- |
| A            | D             |
| B            | E             |
| C            | F             |
| D            | G             |

And it continues like this.

When it reaches the end of the alphabet, it wraps around:

| Plain Letter | Cipher Letter |
| ------------ | ------------- |
| X            | A             |
| Y            | B             |
| Z            | C             |

---

# Encrypting HELLO with Key 3

Let’s encrypt the word:

```text
HELLO
```

Shift each letter **forward by 3**:

| Plaintext Letter | Shifted Letter |
| ---------------- | -------------- |
| H                | K              |
| E                | H              |
| L                | O              |
| L                | O              |
| O                | R              |

So:

```text
HELLO → KHOOR
```

---

# Decrypting KHOOR

Bob receives:

```text
KHOOR
```

To decrypt it, he shifts each letter **backward by 3**:

| Ciphertext Letter | Original Letter |
| ----------------- | --------------- |
| K                 | H               |
| H                 | E               |
| O                 | L               |
| O                 | L               |
| R                 | O               |

So:

```text
KHOOR → HELLO
```

---

# What is Secret in the Caesar Cipher?

In the Caesar cipher:

* The **algorithm** is public: “shift each letter by a certain number”
* The **key** is secret: the number of positions to shift

If an attacker sees the ciphertext `KHOOR` but does not know the key, they must guess the shift.

Since the alphabet has only 26 letters, there are only 25 meaningful shifts.

That means Caesar cipher is extremely weak and easy to break.

---

# Why Caesar Cipher Is Not Secure

The Caesar cipher is useful for learning, but terrible for real security because:

* It has very few possible keys
* A computer can try all possible shifts almost instantly
* Patterns in the language are easy to spot
* It offers almost no real protection against modern attackers

Real cryptographic systems use much stronger algorithms such as **AES (Advanced Encryption Standard)**.

---

# What is Symmetric Encryption?

The Caesar cipher is an example of **symmetric encryption**.

## Definition

**Symmetric encryption** is a type of encryption where the **same key** is used for both:

* **Encryption**
* **Decryption**

In simple words:

> The same secret key that locks the message is also used to unlock it.

---

# How Symmetric Encryption Works

## Step-by-Step

1. Alice and Bob both have the same secret key.
2. Alice uses that key to encrypt her plaintext.
3. The plaintext becomes ciphertext.
4. Alice sends the ciphertext over the network.
5. Bob receives the ciphertext.
6. Bob uses **the same key** to decrypt it.
7. Bob recovers the original plaintext.

---

# Symmetric Encryption Flow

```text
Alice's Plaintext
       ↓
Encryption using Secret Key
       ↓
Ciphertext
       ↓
Sent across network
       ↓
Bob receives Ciphertext
       ↓
Decryption using the SAME Secret Key
       ↓
Original Plaintext
```

---

# Key Rule of Symmetric Encryption

## Same Key for Both Sides

Both the sender and receiver must know the **same secret key**.

If the attacker gets the key, they can decrypt every message protected with it.

So the security of symmetric encryption depends heavily on **keeping the key secret**.

---

# Benefits of Symmetric Encryption

Symmetric encryption is widely used because it is:

## 1. Fast

It can encrypt and decrypt large amounts of data quickly.

## 2. Efficient

It is excellent for:

* Encrypting files
* Encrypting hard drives
* Protecting network traffic
* Securing stored backups
* Encrypting large databases

Because it is fast, symmetric encryption is heavily used in real-world systems.

---

# The Big Problem: Key Distribution

Symmetric encryption has one major weakness:

## How do Alice and Bob safely share the secret key in the first place?

If Alice sends the key openly across the internet, an attacker can steal it.

Then the attacker can read every encrypted message.

This problem is called the **key distribution problem**.

---

# Why This Is a Problem

Imagine Alice wants to send Bob the key.

If she sends the key as plain text, anyone listening can steal it.

You might think:

> “Just encrypt the key too.”

But then you need **another key** to encrypt that key.

And then you need another key for that key.

This creates a loop.

That is why securely sharing the key is one of the biggest challenges of symmetric encryption when used by itself.

---

# Real-World Symmetric Encryption

The Caesar cipher is only for learning. Real systems use modern symmetric algorithms such as:

* **AES (Advanced Encryption Standard)**

These real algorithms are far more secure, but they still follow the same basic principle:

```text
Plaintext + Algorithm + Secret Key → Ciphertext
```

and

```text
Ciphertext + Secret Key → Plaintext
```

---

# Symmetric Encryption Summary Table

| Concept              | Meaning                                             |
| -------------------- | --------------------------------------------------- |
| Plaintext            | Original readable message                           |
| Ciphertext           | Scrambled unreadable message                        |
| Key                  | Secret value used for encryption and decryption     |
| Algorithm            | Public method used to transform the message         |
| Symmetric Encryption | Uses the **same key** for encryption and decryption |

---

# Key Concepts

* **Plaintext** is readable data.
* **Ciphertext** is scrambled unreadable data.
* A **key** is the secret value used to control encryption and decryption.
* An **algorithm** is the public method used to transform data.
* In **symmetric encryption**, the same key is used to both encrypt and decrypt data.
* Symmetric encryption is **fast** and **efficient**.
* The main weakness is the **key distribution problem**.

---

# Summary

Symmetric encryption is one of the core building blocks of cryptography.

It works by using **one shared secret key** to both:

* Encrypt a message
* Decrypt the same message

### Main Idea

```text
Same key locks the message
Same key unlocks the message
```

The Caesar cipher is a simple example of this concept:

* Plaintext: `HELLO`
* Key: `3`
* Ciphertext: `KHOOR`

Although Caesar cipher is not secure, it helps us understand the basic idea of symmetric encryption.

In real life, stronger algorithms like **AES** are used instead.

The biggest challenge of symmetric encryption is safely sharing the secret key between the sender and receiver.
