# Introduction to Cryptography

## Overview

When you visit a website and see the **padlock icon** in your browser, it usually means your connection is protected using cryptography.

But what does that actually mean?

When data travels across the internet, it does **not** move directly from your computer to the destination. Instead, it passes through many systems such as routers, servers, and network devices along the way.

Without protection, someone who can access those systems or intercept the traffic may be able to:

* **Read** your private data
* **Modify** your data before it reaches the destination
* **Block** access to your data or service

Cryptography helps solve these problems by transforming readable information into protected data that unauthorized people cannot easily understand or change.

---

# Why Cryptography Matters

In cyber security, one of the most important goals is protecting digital information.

From the **CIA Triad**, we learned that security focuses on three major principles:

* **Confidentiality** → keeping data secret from unauthorized people
* **Integrity** → ensuring data is not changed without permission
* **Availability** → ensuring data and services are accessible when needed

Cryptography plays a major role in supporting the first two:

* **Confidentiality** by hiding the contents of data
* **Integrity** by helping detect tampering or unauthorized changes

---

# The Main Problem

Imagine you are sending a message over the internet.

You might assume the message travels directly from:

```text id="mq0qol"
Your Device → Recipient
```

But in reality, it often travels through many systems:

```text id="j8jlwm"
Your Device → Router → ISP → Network Devices → Server → Recipient
```

At any point along this path, someone with access to the traffic might try to:

* Read the data
* Change the data
* Steal credentials or sensitive information

If the data is sent in plain form, it becomes easy for attackers to view or manipulate it.

---

# A Real-World Scenario

Imagine you run a small medical clinic.

You need to send patient records to specialists or insurance companies over the internet.

These records may contain highly sensitive information such as:

* Patient names
* Medical conditions
* Treatment history
* Insurance details

If this information is sent without protection, an attacker who intercepts the traffic could read it, steal it, or even change it.

That could lead to:

* Privacy violations
* Legal consequences
* Loss of trust
* Incorrect medical decisions if data is modified

This is where cryptography becomes essential.

---

# What is Cryptography?

**Cryptography** is the practice of protecting information using mathematical techniques so that only authorized people can access or verify it.

In simple words:

> Cryptography transforms normal readable information into a protected form so that unauthorized people cannot easily understand or alter it.

---

# Plaintext and Ciphertext

To understand cryptography, we need two basic terms:

## Plaintext

**Plaintext** is the original readable information before protection is applied.

Example:

```text id="32vifm"
Patient: Ali Khan
Diagnosis: High blood pressure
```

This is normal readable text.

---

## Ciphertext

**Ciphertext** is the scrambled, unreadable version of the original message after cryptographic protection is applied.

Example:

```text id="a66mgh"
X8#fQ2!zM9@LkP$1...
```

This looks like gibberish to anyone who does not have the correct key or method to reverse it.

---

# Simple Idea Behind Cryptography

At a very basic level, cryptography works like this:

1. Start with readable data (**plaintext**)
2. Apply a method to scramble it
3. Produce unreadable protected data (**ciphertext**)
4. Only authorized users with the correct secret can convert it back into readable form

---

# Basic Flow of Cryptography

```text id="adgdaf"
Plaintext → Encryption → Ciphertext → Decryption → Plaintext
```

### Meaning

* **Encryption** = converting readable data into protected unreadable data
* **Decryption** = converting protected unreadable data back into readable data

---

# Everyday Analogy: Locked Box

Imagine you want to send a secret letter to a friend.

If you put the letter in a normal envelope, anyone who gets hold of it can open and read it.

Instead, you place the letter inside a **locked box**.

Only your friend has the correct key to open that box.

### In this analogy:

| Real World      | Cryptography Equivalent |
| --------------- | ----------------------- |
| Letter          | Plaintext               |
| Locked box      | Encryption              |
| Box with lock   | Ciphertext              |
| Key to open box | Decryption key          |

This is the basic idea behind cryptography: **protect the message so only the right person can read it**.

---

# What Cryptography Protects

Cryptography is mainly used to protect:

## 1. Confidentiality

It prevents unauthorized people from reading data.

Example:

* Passwords sent to a website
* Medical records
* Banking information
* Private messages

If intercepted, encrypted data should remain unreadable to attackers.

---

## 2. Integrity

Cryptography can also help detect whether data has been changed.

Example:

If a message says:

```text id="e2gls8"
Transfer $1000 to Account A
```

and an attacker changes it to:

```text id="w6omag"
Transfer $1000 to Account B
```

integrity has been broken.

Cryptographic techniques can help detect such tampering.

---

# What Cryptography Does Not Automatically Solve

Cryptography is powerful, but it is **not magic**.

It does **not automatically guarantee availability**.

For example:

* A website can still be taken offline by an attack
* A server can still crash
* A service can still become unreachable

So while cryptography is essential for **confidentiality** and **integrity**, availability often requires other protections too.

This helps protect data such as:

* Login credentials
* Payment information
* Messages submitted through forms
* Session data

It reduces the chance that someone on the network can read or modify your communication.
