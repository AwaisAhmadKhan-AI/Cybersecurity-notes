# Key Terminology

# Scope

The boundaries and rules of an authorized security assessment.

Scope defines:

* What systems may be tested
* What actions are permitted
* What is prohibited

Testing outside scope is unauthorized.

---

# Vulnerability

A weakness or flaw that attackers may exploit.

Examples:

* Weak passwords
* Exposed admin panels
* Misconfigured permissions

---

# Exploit

A technique used to abuse a vulnerability.

Examples:

* Bypassing authentication
* Executing malicious code
* Accessing restricted data

---

# Enumeration

The process of gathering information about a target.

Enumeration may identify:

* Users
* Services
* Directories
* Open ports
* Hidden resources

It is often one of the first stages of an attack.

---

# Credentials

Authentication details used to access systems.

Examples:

* Usernames
* Passwords
* API keys
* Tokens

Compromised credentials are extremely valuable to attackers.

---

# Authentication

The process of verifying identity.

Authentication systems confirm whether:

* A user is legitimate
* Login credentials are correct

Examples include:

* Passwords
* MFA codes
* Biometrics

---

# Dictionary Attack

A password attack that uses a predefined list of possible passwords.

Instead of guessing randomly, attackers test:

* Common passwords
* Frequently reused credentials
* Leaked password lists
