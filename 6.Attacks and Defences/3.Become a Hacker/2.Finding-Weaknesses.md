# Finding Weaknesses — Offensive Security Basics

# Introduction

Offensive security is the practice of proactively testing systems, applications, and networks to identify weaknesses before real attackers can exploit them.

Instead of waiting for a cyberattack to happen, security professionals simulate attacks in a controlled and authorized manner to improve defenses.

This process helps organizations:

* Discover vulnerabilities
* Improve security posture
* Validate defensive controls
* Reduce real-world risk

---

# Core Offensive Security Terms

# Red Teaming

A structured and authorized attack simulation designed to imitate real-world adversaries.

## Goals

* Test security defenses
* Measure detection capabilities
* Assess incident response readiness

## Characteristics

* Realistic attack scenarios
* Defined objectives
* Often stealth-focused

---

# Penetration Testing

A controlled security assessment where authorized testers attempt to identify and exploit vulnerabilities.

## Purpose

* Understand actual security risks
* Validate weaknesses
* Provide remediation guidance

## Key Point

Penetration testing always occurs within an approved scope.

---

# Vulnerability

A weakness or flaw that could be abused by an attacker.

## Examples

* Weak passwords
* Exposed admin panels
* Misconfigured servers
* Outdated software

---

# Exploit

A technique or method used to take advantage of a vulnerability.

## Examples

* Gaining unauthorized access
* Bypassing authentication
* Extracting sensitive data

---

# Scope

Defines what is allowed during a security engagement.

## Scope Includes

* Authorized systems
* Permitted testing methods
* Time limitations
* Restricted areas

---

# The Importance of Permission

Ethical hacking requires explicit authorization.

Without permission:

* Testing systems is illegal
* Accessing systems can violate laws and policies

Ethical hackers operate:

* Safely
* Legally
* Within defined boundaries

# Practical Scenario

## The Situation

Mike recently developed an online shopping website and wants to ensure no sensitive pages are publicly exposed before launch.

* Assess the website

# Reconnaissance and Enumeration

One of the first stages in offensive security is:

> Enumeration

This involves discovering:

* Hidden directories
* Login pages
* Admin portals
* Backup files
* Sensitive endpoints

Attackers often search for pages that developers forgot to protect.

---


* The page may not exist

# Limitations of Manual Testing

Manual enumeration works for small lists, but becomes inefficient when testing:

* Hundreds of directories
* Thousands of filenames
* Large web applications

This is why automated tools are used.

---

# Using Gobuster

## What is Gobuster?

Gobuster is a command-line tool used for:

* Directory enumeration
* File discovery
* DNS subdomain brute forcing

It automates the process of finding hidden content on web servers.

---

# Gobuster Command

```bash id="ezqj64"
gobuster dir --url http://www.onlineshop.thm/ -w /usr/share/wordlists/dirbuster/directory-list.txt
```

---

# Command Breakdown

## `gobuster`

The main tool being executed.

---

## `dir`

Specifies:

* Directory enumeration mode

Gobuster attempts to discover:

* Hidden directories
* Hidden files

---

## `--url`

Defines the target website.

Example:

```bash id="8u6nkl"
--url http://www.onlineshop.thm/
```

---

## `-w`

Specifies the wordlist used for guessing paths.

Example:

```bash id="jlwm42"
-w /usr/share/wordlists/dirbuster/directory-list.txt
```

The wordlist contains:

* Common directory names
* Common filenames
* Frequently exposed endpoints

---

# What Gobuster Does

Gobuster automatically:

1. Reads words from the wordlist
2. Appends them to the target URL
3. Sends HTTP requests
4. Analyzes responses

Example attempts:

```text id="6nyh3f"
/admin
/login
/uploads
/backup
/config
```

---

# Why Enumeration Matters

Hidden pages may expose:

* Admin dashboards
* Development tools
* Sensitive files
* Misconfigured services

Attackers commonly use enumeration during:

* Reconnaissance
* Initial access attempts

Defenders use the same techniques to identify and secure weaknesses first.

---

# Common Security Risks from Exposed Pages

| Exposed Resource | Risk                               |
| ---------------- | ---------------------------------- |
| `/admin`         | Unauthorized administration access |
| `/backup`        | Sensitive data leakage             |
| `/config`        | Credential exposure                |
| `/login`         | Brute-force attack surface         |
| `/mail`          | Internal communication exposure    |

---

# Ethical Considerations

Enumeration tools must only be used:

* On systems you own
* In labs and training environments
* With explicit authorization

Unauthorized scanning may:

* Trigger alerts
* Violate policies
* Be illegal
