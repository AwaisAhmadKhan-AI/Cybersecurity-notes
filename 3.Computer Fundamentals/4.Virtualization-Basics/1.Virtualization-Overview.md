# Virtualization Overview

## What is Virtualization?

**Virtualization** is a technology that allows multiple virtual computers to run on a single physical server.

Instead of dedicating one physical server to one application, multiple systems can safely share the same hardware.

---

## Before Virtualization

### Traditional Approach:
```
1 Server = 1 Application
```

Examples:
- One server for website
- One server for database
- One server for email
- One server for internal application

---

## Problems with This Approach

### High Cost
- More hardware to buy
- More electricity usage
- More cooling required
- More maintenance

---

### Low Resource Usage
- Most servers only used 5–20% of their capacity
- CPU, RAM, and storage were often wasted

---

### Slow Deployment
- Setting up a new server could take days or weeks

---

### Difficult Scaling
- More demand often meant buying more physical servers

---

## The Solution: Virtualization

Virtualization introduced a new idea:

> Multiple systems can safely share one physical server.

---

## Hypervisor

A **Hypervisor** is software that manages virtualization.

### Main Role:
- Creates Virtual Machines (VMs)
- Allocates CPU, RAM, and storage
- Keeps VMs isolated from each other

👉 Think of it as a manager that controls shared resources.

---

## Building Analogy

### Without Virtualization

Imagine:
- One person lives in an entire 10-floor building
- Uses only one floor
- Pays to maintain the whole building

👉 Most resources are wasted

---

### With Virtualization

The building is divided into apartments:

- Each apartment is private
- Multiple people live independently
- Everyone shares the building infrastructure

👉 Resources are used efficiently

---

## Analogy Mapping

| Real World | Virtualization |
|------------|---------------|
| Building | Physical Server |
| Apartment | Virtual Machine (VM) |
| Tenant | Operating System / Application |
| Building Manager | Hypervisor |

---

## Virtual Machine (VM)

A **Virtual Machine (VM)** is a software-based computer.

Each VM has:
- Its own Operating System
- Its own applications
- Its own settings

Even though multiple VMs share the same hardware.

---

## Benefits of Virtualization

- Better hardware utilization
- Lower costs
- Faster deployment
- Easier scaling
- Multiple systems on one server

---

## Key Takeaway

Virtualization allows multiple **Virtual Machines (VMs)** to run independently on a single physical server using a **Hypervisor**, making computing more efficient, scalable, and cost-effective.
