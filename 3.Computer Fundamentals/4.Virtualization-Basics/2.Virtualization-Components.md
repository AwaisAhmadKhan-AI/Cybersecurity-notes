# Virtualization Components

## Hypervisor (The Building Manager)

### What is a Hypervisor?

A **Hypervisor** is software that creates and manages Virtual Machines (VMs).

Its job is to:

- Create virtual machines
- Allocate CPU, RAM, and storage
- Keep VMs isolated from each other
- Start, stop, clone, pause, and delete VMs

👉 Think of it as the **building manager** of a virtualized system.

---

## Types of Hypervisors

### Type 1 Hypervisor (Bare Metal)

- Runs directly on physical hardware
- Faster and more efficient
- Common in servers and data centers

### Best For:
- Production servers
- Database servers
- Data centers

---

### Type 2 Hypervisor (Hosted)

- Runs on top of an existing operating system
- Easier to install and use
- Great for learning and testing

### Best For:
- Kali Linux labs
- Software testing
- Malware analysis
- Home labs

---

## Examples of Type 2 Hypervisors

- Oracle VirtualBox
- VMware Workstation

---

## Virtual Machines (VMs)

### What is a VM?

A **Virtual Machine (VM)** is a virtual computer created by a hypervisor.

Even though it is virtual, it behaves like a real computer.

---

### A VM Has:

- Virtual CPU
- Virtual RAM
- Virtual Storage
- Virtual Network Card

---

### Key Features

- Can run Windows, Linux, macOS, etc.
- Isolated from other VMs
- One VM crashing does not affect others

---

## Common Uses of VMs

### Run Another Operating System
Example:
- Running Kali Linux on a Windows PC

### Malware Testing
- Safely analyze suspicious files
- Protect the main system

---

## Containers (The Rooms Inside the Apartment)

### What is a Container?

A **Container** is a lightweight environment that contains:

- An application
- Required libraries
- Dependencies

Unlike VMs, containers do NOT include a full operating system.

---

## How Containers Work

Containers share the host operating system's:

- Kernel
- Resources

Because of this:

- Start very quickly
- Use less RAM and CPU
- Are more lightweight than VMs

---

## Benefits of Containers

- Fast deployment
- Low resource usage
- Consistent environment
- Easy scaling
- Isolation between applications

---

## Important Limitation

Containers must match the host operating system type.

Example:
- Linux host → Linux containers
- Windows host → Windows containers

❌ Cannot run Windows containers on Linux

---

## Docker

### What is Docker?

**Docker** is the most popular platform for creating and managing containers.

It makes:
- Building applications
- Deploying applications
- Running containers

much easier.

---

## VM vs Container

| Virtual Machine (VM) | Container |
|---------------------|-----------|
| Includes full OS | Shares host OS |
| Uses more resources | Uses fewer resources |
| Slower startup | Starts almost instantly |
| Better isolation | Lightweight and scalable |
| More flexible | Faster deployment |

---

## Simple Analogy

| Real World | Virtualization |
|------------|---------------|
| Building | Physical Server |
| Building Manager | Hypervisor |
| Apartment | Virtual Machine |
| Room inside apartment | Container |

---

## Key Takeaway

- **Hypervisor** creates and manages VMs.
- **VMs** are complete virtual computers with their own OS.
- **Containers** are lightweight environments that share the host OS.
- **Docker** is a popular tool used to run containers.
