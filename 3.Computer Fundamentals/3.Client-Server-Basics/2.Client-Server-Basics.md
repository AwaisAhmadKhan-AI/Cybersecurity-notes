## Client–Server Basics: Core Components

A simple way to understand how computers talk to each other is to compare it with ordering food from a takeaway shop.

---

## Service, Client, Server

In the analogy:

- **Service** → The pizza shop (what is being provided)
- **Client** → Alice (the one who requests something)
- **Server** → Luigi’s Pizza system (the one that provides the service)

### In computer terms:
- A **client** (e.g., browser) *requests* something
- A **server** *provides* that service (webpages, files, data)

👉 Important rule:  
The **client always initiates the request**.

---

## Request and Response

- Alice places an order → this is a **request**
- Luigi’s Pizza replies with food or an error → this is a **response**

### In computing:
- Client sends a **request** (e.g., “GET webpage”)
- Server sends a **response**:
  - Success (data returned)
  - Error (resource not found / request invalid)

---

## Protocol

A **protocol** is the set of rules for communication (like a shared language).

In the analogy:
- Bob must understand Luigi’s ordering system
- Without the correct “language,” the order fails

### In computer systems, a protocol defines:
- What commands are allowed (e.g., GET, POST)
- How requests are structured
- How responses should look
- What to do when something goes wrong

👉 Example protocols: HTTP, HTTPS, FTP

---

## Port

A **port** is like a specific door for a specific service.

### Analogy:
Luigi’s Pizza has multiple doors:
- Door A → Takeaway
- Door B → Dine-in
- Door C → Delivery

### In computing:
- A server can run multiple services at once
- Each service listens on a **specific port**
- Clients must connect to the correct port to reach the service

👉 Example:
- Web traffic → Port 80 (HTTP)
- Secure web → Port 443 (HTTPS)

---

## DNS (Domain Name System)

Bob doesn’t know the exact location of Luigi’s Pizza, so he uses GPS.

### In computing:
- Humans use domain names (e.g., google.com)
- Computers use **IP addresses** (exact location of a server)

### DNS does:
- Converts domain names → IP addresses
- Works like a **phonebook or GPS for the internet**

👉 Example:
- `google.com` → `142.250.x.x`

---

## Final Idea

- Client = requests  
- Server = responds  
- Protocol = rules of communication  
- Port = service entry door  
- DNS = name → address translator  
- Request/Response = communication cycle
