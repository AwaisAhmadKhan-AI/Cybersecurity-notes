## Web Communication in Practice (HTTP/HTTPS)

HTTP (HyperText Transfer Protocol) and HTTPS (secure version) are **stateless client–server protocols** used for web communication.

---

## Stateless Nature of HTTP

HTTP is **stateless**, meaning:
- Each request is handled independently
- The server does NOT remember previous requests

### Problem this creates:
If a user logs in:
- Without state → user must log in again on every request

### Solution (Statefulness added on top):
Web apps use:
- **Sessions**
- **Cookies**
- **Tokens**

These allow the server to “remember” users across requests.

---

## HTTP Methods (Commands)

HTTP defines standard actions called **methods** (not commands). There are 9 total, but the main ones are:

- GET → retrieve data
- POST → send data / create
- PUT → update data
- DELETE → remove data
- PATCH → partial update
- HEAD → headers only (no body)
- OPTIONS → supported methods info
- CONNECT → tunnel connection
- TRACE → debugging request path

👉 Most common in real usage: **GET + POST**

---

## GET Method (Most Important)

The **GET method** is used to retrieve data from a server.


When you type a URL in a browser:
- The browser automatically builds a GET request
- Sends it to the web server
- Receives a response (HTML, images, etc.)

---

## HTTP Request Flow

1. Browser sends GET request
2. Server receives request
3. Server processes it
4. Server sends back response
5. Browser displays the result

---

## HTTP Request Components (Observed in Dev Tools)

When inspecting a request, you will see:

- **Scheme** → HTTP or HTTPS protocol used
- **Host** → Target server domain/name
- **Filename / Path** → Resource requested (e.g., `/` = homepage)
- **Address** → IP address of the server (e.g., `127.0.0.1` for local host)
- **Status Code** → Result of request (e.g., `200 OK`)

---

## HTTP Response Structure

A response has two parts:

### 1. Response Header
Contains metadata:
- Status code
- Content type
- Server info
- etc.

### 2. Response Body
Contains the actual data:
- HTML page
- Images
- JSON data
- etc.

---

## Key Idea

- HTTP = request/response system
- Browser = client
- Server = responder
- Cookies/sessions = memory layer on top of stateless HTTP
