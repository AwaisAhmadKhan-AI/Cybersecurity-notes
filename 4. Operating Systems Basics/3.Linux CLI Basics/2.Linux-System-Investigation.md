# Linux System Investigation (Basic System Information Gathering)

## Overview

In this section, you learned how to inspect a Linux system and gather essential information such as:

- Current user
- Kernel version
- Disk usage
- Operating system details

This process is often called a **system reconnaissance check**, commonly used in cybersecurity to understand the environment.

---

# Why System Information Matters

System information helps security teams:

- Identify the operating system
- Understand system capabilities
- Check available storage
- Detect misconfigurations
- Prepare for analysis or testing

Think of it as a **health check of the machine**.

---

# Step 1: Identify Current User

## Command: whoami

### Purpose
Shows the username of the currently logged-in user.

### Usage

```bash
whoami
```

### Example Output

```bash
ubuntu
```

### Meaning

You are logged in as the user: `ubuntu`

---

# Step 2: System and Kernel Information

## Command: uname -a

### Purpose
Displays detailed system information including kernel version and architecture.

### Usage

```bash
uname -a
```

### Example Output

```bash
Linux tryhackme 6.x.x-aws #17-Ubuntu SMP x86_64 GNU/Linux
```

---

## Breakdown

| Field | Meaning |
|------|--------|
| Linux | Kernel type |
| hostname | Machine name |
| kernel version | OS kernel build |
| x86_64 | 64-bit architecture |
| GNU/Linux | OS environment |

---

## Simpler Version

```bash
uname
```

Output:

```bash
Linux
```

(Only shows kernel name)

---

# Step 3: Check Disk Usage

## Command: df -h

### Purpose
Shows disk space usage in a human-readable format.

### Usage

```bash
df -h
```

---

## Key Columns

| Column | Meaning |
|--------|--------|
| Size | Total disk space |
| Used | Space used |
| Avail | Free space |
| Use% | Percentage used |
| Mounted on | Location in filesystem |

---

## Key Insight

- `/dev/root` → main system disk
- `tmpfs` → temporary memory-based storage (RAM)
- Used to monitor system health and storage limits

---

# Step 4: System Configuration Files

## Location: /etc Directory

### Purpose
Stores system configuration files.

### Command

```bash
cd /etc
ls
```

---

## Example Files

- `hosts`
- `fstab`
- `os-release`
- `network configuration files`
- `system settings`

---

# Reading OS Information File

## Command: cat os-release

### Purpose
Displays Linux distribution details.

### Usage

```bash
cat /etc/os-release
```

---

## Example Output

```bash
PRETTY_NAME="Ubuntu 24.04.1 LTS"
NAME="Ubuntu"
VERSION_ID="24.04"
```

---

## Key Information

- Distribution name: Ubuntu
- Version: 24.04.1 LTS
- Codename: Noble Numbat

---

# Core System Information Commands

| Command | Purpose |
|--------|--------|
| whoami | Show current user |
| uname -a | Full system info |
| uname | Kernel name only |
| df -h | Disk usage |
| cd /etc | System config directory |
| cat os-release | OS distribution details |

# Key Takeaway

Linux system investigation is about combining simple commands to understand:

- Who you are on the system
- What system you are using
- How resources are being used
- What configuration defines the OS

These fundamentals form the base of **cybersecurity system analysis and forensics**.
