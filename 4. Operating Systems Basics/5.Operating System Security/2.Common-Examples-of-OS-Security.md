# Common Examples of Operating System Security

## Overview

Operating System Security focuses on protecting the three security goals:

- **Confidentiality** → Keep data private
- **Integrity** → Keep data accurate and unchanged
- **Availability** → Keep systems accessible

Common security weaknesses that attackers target include:

1. Weak Passwords
2. Weak File Permissions
3. Malicious Programs

---

# 1. Authentication and Weak Passwords

## What is Authentication?

Authentication is the process of verifying your identity.

### Three Types of Authentication

### Something You Know

- Password
- PIN

### Something You Are

- Fingerprint
- Face recognition

### Something You Have

- Mobile phone
- Security token
- SMS verification code

---

## Why Passwords Are a Target

Passwords are the most common authentication method and therefore the most frequently attacked.

Many users:

- Use simple passwords
- Reuse passwords on multiple accounts
- Use personal information

### Bad Examples

- Birth dates
- Pet names
- Phone numbers
- Family names

Attackers often guess these first.

---

## Common Weak Passwords

| Rank | Password |
|--------|----------|
| 1 | 123456 |
| 2 | 123456789 |
| 3 | qwerty |
| 4 | password |
| 5 | 111111 |
| 6 | 12345678 |
| 7 | abc123 |
| 8 | 1234567 |
| 9 | password1 |
| 10 | 12345 |

### Why These Are Weak

- Easy to guess
- Very common
- Based on keyboard patterns
- Found in password dictionaries

Examples:

```text
123456
password
qwerty
iloveyou
dragon
```

---

## Risks of Weak Passwords

If an attacker guesses your password, they may gain access to:

- Email accounts
- Social media accounts
- Online banking
- Personal files

---

## Best Practices

✔ Use long passwords

✔ Use a mix of:

- Uppercase letters
- Lowercase letters
- Numbers
- Symbols

✔ Use different passwords for different accounts

✔ Enable Multi-Factor Authentication (MFA)

---

# 2. Weak File Permissions

## What Are File Permissions?

File permissions determine:

```text
Who can access a file?
Who can modify a file?
Who can delete a file?
```

---

## Principle of Least Privilege

Users should only have the permissions necessary to perform their tasks.

### Example

If you're planning a family trip:

- Family members should access trip files.
- Strangers should not.

---

## Why Weak Permissions Are Dangerous

### Confidentiality Risk

Attackers may read files they shouldn't access.

Examples:

- Personal documents
- Password files
- Company data

---

### Integrity Risk

Attackers may modify files they shouldn't change.

Examples:

- Change important settings
- Alter documents
- Replace files with malicious versions

---

# 3. Malicious Programs (Malware)

## What is Malware?

Malware is software designed to harm systems or users.

Examples include:

- Viruses
- Worms
- Trojans
- Ransomware

---

## Trojan Horse

### What It Does

A Trojan appears legitimate but secretly helps attackers.

### Impact

Attackers may:

- Access your system
- Read files
- Steal information
- Modify data

### Security Goals Affected

- Confidentiality
- Integrity

---

## Ransomware

### What It Does

Ransomware encrypts files and makes them inaccessible.

### Example

Before Encryption:

```text
VacationPhotos.jpg
Assignment.docx
```

After Encryption:

```text
Unreadable encrypted data
```

---

### Attacker's Goal

The attacker demands money (ransom) in exchange for the decryption key.

### Security Goal Affected

- Availability

Because users can no longer access their files.

---

# Security Impact Summary

| Threat | Confidentiality | Integrity | Availability |
|----------|----------------|------------|-------------|
| Weak Passwords | ✔ | ✔ | ✖ |
| Weak File Permissions | ✔ | ✔ | ✖ |
| Trojan Horse | ✔ | ✔ | ✖ |
| Ransomware | ✖ | ✖ | ✔ |

---

# Key Takeaway

The three most common OS security weaknesses are:

1. **Weak Passwords** → Easy account compromise.
2. **Weak File Permissions** → Unauthorized file access or modification.
3. **Malware** → Can steal data, modify files, or block access.

Always follow the **Principle of Least Privilege**, use **strong passwords**, and avoid running **untrusted programs**.
