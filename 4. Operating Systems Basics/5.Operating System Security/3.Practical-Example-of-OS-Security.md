# Practical Example of OS Security

## Overview

This practical example demonstrates how weak passwords and poor security practices can allow an attacker to gain access to a system.

### Attack Scenario

An attacker:

1. Finds a username.
2. Guesses the password.
3. Gains access to the system.
4. Attempts privilege escalation to become an administrator.

---

# Important Accounts

## Linux

Highest privilege account:

```text
root
```

Has complete control over the system.

---

## Windows

Highest privilege account:

```text
Administrator
```

Has complete unrestricted access.

---

# Attack Scenario

During a security assessment, a sticky note was discovered containing:

```text
Username: sammie
Password: dragon
```

This demonstrates a common security mistake:

❌ Writing passwords on sticky notes.

---

# SSH (Secure Shell)

## What is SSH?

SSH allows secure remote login to another system.

### Syntax

```bash
ssh username@IP_Address
```

### Example

```bash
ssh sammie@10.112.180.29
```

The system will ask for the password.

---

# First-Time SSH Connection

When connecting for the first time:

```text
Are you sure you want to continue connecting (yes/no)?
```

Type:

```text
yes
```

This saves the server's SSH key.

---

# Password Entry in SSH

When typing a password:

```text
Nothing appears on screen.
```

No:

- Stars (*)
- Dots (•)
- Characters

This is normal behavior.

# Verify Current User

## Command: whoami

### Purpose

Displays the currently logged-in user.

### Syntax

```bash
whoami
```

### Example

```bash
sammie@machine:~$ whoami

sammie
```

---

# List Files

## Command: ls

### Purpose

Lists files and folders in the current directory.

### Syntax

```bash
ls
```

### Example

```bash
sammie@machine:~$ ls

country.txt
draft.md
icon.png
password.txt
profile.jpg
```

---

# Read File Contents

## Command: cat

### Purpose

Displays the contents of a text file.

### Syntax

```bash
cat filename
```

### Example

```bash
cat draft.md
```

Output:

```text
# Operating System Security

Reusing passwords means that your password
for other sites becomes exposed if one
service is hacked.
```

---

# View Command History

## Command: history

### Purpose

Displays previously executed commands.

### Syntax

```bash
history
```

### Example

```bash
history
```

This can reveal:

- Previous commands
- User mistakes
- Sensitive information

# Other Users on the System

Known usernames:

```text
johnny

```

# Switching Users

## Command: su

### Purpose

Switch to another user account.

### Syntax

```bash
su - username
```

### Example

```bash
su - johnny
```

The system will request Johnny's password.

Example:

```text

```

Problems:

```text
One breached account
```

# Attack Flow Summary

```text
Find Username
      ↓
Guess Password
      ↓
Login via SSH
      ↓
Verify User (whoami)
      ↓
Explore Files (ls)
      ↓
Read Files (cat)
      ↓
Check History (history)
      ↓
Attempt Privilege Escalation
```

---

# Security Lessons

### Never

❌ Write passwords on sticky notes

❌ Use weak passwords

❌ Reuse passwords across accounts

---

### Always

✔ Use strong unique passwords

✔ Use password managers

✔ Enable Multi-Factor Authentication (MFA)

✔ Follow the Principle of Least Privilege

---

# Key Takeaway

This exercise demonstrates how a simple weak password can allow an attacker to:

1. Gain system access.
2. Read sensitive files.
3. Explore the system.
4. Potentially escalate privileges.

Strong passwords and proper security practices are essential for protecting operating systems.
