# Introduction to Operating System Security

## What is an Operating System (OS)?

An **Operating System (OS)** is software that sits between the **hardware** and the **applications** you use.

It manages hardware resources and allows programs to run properly.

### Examples of Operating Systems

- Windows
- Linux
- macOS
- Android
- iOS
- Chrome OS

---

# Hardware vs Software

## Hardware

Physical parts of a computer that you can touch.

### Examples

- Keyboard
- Mouse
- Monitor
- Printer
- CPU
- RAM
- HDD/SSD

## Software

Programs and applications that run on the computer.

### Examples

- Chrome
- Firefox
- WhatsApp
- Telegram
- Microsoft Word

---

# Why Do We Need an Operating System?

Applications cannot communicate directly with hardware.

The OS acts as a bridge between:

```text
Applications
      ↓
Operating System
      ↓
Hardware
```

The OS:

- Controls hardware
- Manages memory
- Runs applications
- Handles files
- Enforces security rules

# Why Operating System Security Matters

Modern devices store a lot of personal and sensitive data.

### On Smartphones

- Private chats
- Photos and videos
- Emails
- Banking apps
- Saved passwords

### On Computers

- University or work files
- Personal documents
- Passport/ID copies
- Emails
- Saved browser passwords
- Photos and videos

Without security, anyone could access or steal this information.

---

# The CIA Triad (Core Security Goals)

Operating system security focuses on protecting three things:

## 1. Confidentiality 🔒

Only authorized people can access data.

### Examples

- Password-protected accounts
- File permissions
- Encryption

**Goal:** Keep information private.

---

## 2. Integrity ✔️

Data should not be changed without permission.

### Examples

- Protecting files from modification
- Detecting tampered data
- Checksums and digital signatures

**Goal:** Keep data accurate and trustworthy.

---

## 3. Availability 🟢

Systems and data should be available whenever needed.

### Examples

- Reliable operating system
- Backups
- Protection against crashes and attacks

**Goal:** Ensure the system works when users need it.

---

# Simple Example

Imagine your laptop contains:

- University assignments
- Personal photos
- Bank information
- Saved passwords

You want:

- Nobody else to see them (**Confidentiality**)
- Nobody to modify them (**Integrity**)
- Access whenever you need them (**Availability**)

This is why operating system security is important.

---

# Key Takeaway

An **Operating System** manages hardware and software while protecting your data.

The three main security goals are:

1. **Confidentiality** → Keep data private.
2. **Integrity** → Keep data accurate.
3. **Availability** → Keep systems accessible.

Together, these form the foundation of **Operating System Security**.
