# Windows CLI Basics: Navigating Files & Finding Files

## Overview

The Windows Command Prompt (CLI) allows users to interact with the operating system using text commands instead of the graphical interface.

These skills are useful for:

- System administration
- Troubleshooting
- Cybersecurity
- Automation

---

# Why Use the Command Line?

Advantages:

- Faster than GUI navigation
- More control over the system
- Required by many cybersecurity tools
- Useful for automation and scripting

---

# Essential Windows CLI Commands

## 1. cd (Current Directory)

### Purpose

Displays your current location in the filesystem.

### Command

```cmd
cd
```

### Example Output

```cmd
C:\Users\Administrator
```

### Remember

👉 "Where am I?"

---

# Viewing Files and Folders

## 2. dir

### Purpose

Lists files and folders in the current directory.

### Command

```cmd
dir
```

### Example

```cmd
Documents
Downloads
Desktop
Pictures
```

---

## Information Displayed

- Files
- Folders
- Dates
- Sizes

---

# Viewing Hidden Files

## 3. dir /a

### Purpose

Displays all files, including hidden files and folders.

### Command

```cmd
dir /a
```

### Why Use It?

Some important files may be hidden by Windows.

---

## Note

Hidden ≠ Secret

Windows simply hides certain files by default.

---

# Navigating Directories

## 4. cd FolderName

### Purpose

Move into a folder.

### Example

```cmd
cd Documents
```

Moves into:

```cmd
C:\Users\Administrator\Documents
```

---

## 5. cd ..

### Purpose

Move back one directory level.

### Example

```cmd
cd ..
```

---

# Finding Files

## 6. dir /s filename

### Purpose

Searches for a file in all subdirectories.

### Syntax

```cmd
dir /s filename
```

### Example

```cmd
dir /s task_brief.txt
```

---

## What Does /s Do?

```cmd
/s
```

Searches recursively through all folders below the current directory.

---

## Result

Returns the full path of the file.

Example:

```cmd
C:\Users\Administrator\Projects\task_brief.txt
```

---

# Navigating to the File

Once the path is known:

### Example

```cmd
cd C:\Users\Administrator\Projects
```

---

### Verify File Exists

```cmd
dir
```

Output should show:

```cmd
task_brief.txt
```

---

# Reading Files

## 7. type

### Purpose

Displays the contents of a text file.

### Syntax

```cmd
type filename
```

### Example

```cmd
type task_brief.txt
```

---

## Result

Prints file contents directly in Command Prompt.

---

# Commands Learned

| Command | Purpose |
|----------|----------|
| cd | Show current directory |
| cd folder | Move into folder |
| cd .. | Move back one directory |
| dir | List files and folders |
| dir /a | Show hidden files |
| dir /s filename | Search for a file |
| type filename | Display file contents |

---

# Linux vs Windows Equivalent

| Linux | Windows |
|---------|---------|
| pwd | cd |
| ls | dir |
| ls -al | dir /a |
| cd | cd |
| find | dir /s |
| cat | type |

---

# Key Takeaway

The core Windows CLI commands for file navigation are **cd**, **dir**, **dir /a**, **dir /s**, and **type**. Using these commands, you can navigate directories, discover hidden files, locate files anywhere on the system, and read file contents directly from the command line.
