# Windows CLI: Gathering System Information

## Overview

System information gathering is one of the first tasks performed by IT administrators and cybersecurity professionals.

Before troubleshooting or investigating a system, important questions include:

- Who am I logged in as?
- What is this computer called?
- Which Windows version is installed?
- How is the machine connected to the network?

---

# Why Gather System Information?

System information helps with:

- Troubleshooting issues
- Security investigations
- Incident response
- Network management
- Asset identification

---

# 1. Identify Current User

## Command: whoami

### Purpose

Displays the username of the currently logged-in account.

### Syntax

```cmd
whoami
```

### Example

```cmd
C:\> whoami

administrator
```

### Why It Matters

Different users have different permissions.

This command helps determine:

- Current account
- User privileges
- Ownership of actions on the system

---

# 2. Find the Computer Name

## Command: hostname

### Purpose

Displays the computer's hostname.

### Syntax

```cmd
hostname
```

### Example

```cmd
C:\> hostname

WORKSTATION-01
```

### Why It Matters

A hostname:

- Identifies a machine on a network
- Helps administrators locate systems
- Assists in network troubleshooting

---

# 3. View Operating System Information

## Command: systeminfo

### Purpose

Displays detailed information about Windows and system hardware.

### Syntax

```cmd
systeminfo
```

### Example

```cmd
C:\> systeminfo
```

---

## Important Fields

### OS Name

Shows the installed operating system.

Example:

```cmd
OS Name: Microsoft Windows Server 2019
```

---

### OS Version

Shows the Windows version and build number.

Example:

```cmd
OS Version: 10.0.17763
```

---

### System Type

Shows whether Windows is 32-bit or 64-bit.

Example:

```cmd
System Type: x64-based PC
```

---

## Additional Information Available

- Host Name
- Installation Date
- Processor Information
- RAM Information
- Domain Information
- Network Adapter Details

---

# 4. View Network Information

## Command: ipconfig

### Purpose

Displays network configuration information.

### Syntax

```cmd
ipconfig
```

### Example

```cmd
C:\> ipconfig
```

---

## Important Fields

### IPv4 Address

The machine's local network address.

Example:

```cmd
IPv4 Address . . . . . . . : 192.168.1.25
```

### Purpose

- Identifies the device on the network
- Used for communication between devices

---

### Default Gateway

The router used to access other networks and the Internet.

Example:

```cmd
Default Gateway . . . . . : 192.168.1.1
```

### Purpose

- Sends traffic outside the local network
- Connects the system to the Internet

```cmd
whoami
```

```cmd
hostname
```

```cmd
systeminfo
```

### Check Network Configuration


# Why Cybersecurity Professionals Use These Commands

These commands help analysts quickly determine:

- Who is using the system
- What machine they are investigating
- Which operating system is installed
- How the system is connected to the network

This information forms the foundation of system enumeration and incident investigation.

---

# Key Takeaway

The most important Windows information-gathering commands are:

```cmd
whoami
hostname
systeminfo
ipconfig
```

Together, they reveal the user identity, computer name, operating system details, and network configuration of a Windows machine.
