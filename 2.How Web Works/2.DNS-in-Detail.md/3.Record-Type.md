# DNS Record Types

## Overview

DNS doesn’t only map websites to IP addresses — it uses different **record types** to handle different kinds of requests.

---

## 1. A Record

### What it does:
Maps a domain to an **IPv4 address**

### Example:
```
example.com → 104.26.10.229
```

👉 Used for standard internet addresses (IPv4)

---

## 2. AAAA Record

### What it does:
Maps a domain to an **IPv6 address**

### Example:
```
example.com → 2606:4700:20::681a:be5
```

👉 Used for newer IP format (IPv6)

---

## 3. CNAME Record

### What it does:
Maps a domain to **another domain name**

### Example:
```
store.tryhackme.com → shops.shopify.com
```

👉 Then DNS looks up the second domain to get the IP

---

## 4. MX Record

### What it does:
Directs **email traffic** to mail servers

### Example:
```
tryhackme.com → mail server (e.g. google mail servers)
```

---

### Important Feature:
- Has **priority values**
- Lower number = higher priority

👉 Used for backup mail servers if main one fails

---

## 5. TXT Record

### What it does:
Stores **text-based information**

---

### Common Uses:

#### 1. Email Security
- Prevents spam and email spoofing
- Example SPF record:
```
v=spf1 ip4:192.0.2.0/24 include:_spf.google.com ~all
```

---

#### 2. Domain Verification
- Confirms ownership when using third-party services

Example:
```
MS=ms12345678
```

---

#### 3. Security Policies (DMARC)
Example:
```
v=DMARC1; p=reject; rua=mailto:reports@example.com
```

---

## Simple Summary

| Record Type | Purpose |
|------------|--------|
| A | IPv4 address |
| AAAA | IPv6 address |
| CNAME | Domain to domain mapping |
| MX | Email servers |
| TXT | Text / verification / security |

---

## Key Takeaway

DNS record types define **how a domain behaves**, not just where it points — including websites, emails, and security settings.
