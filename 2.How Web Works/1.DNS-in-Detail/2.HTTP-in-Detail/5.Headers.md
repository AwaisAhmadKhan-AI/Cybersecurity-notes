# HTTP Headers

## Overview

HTTP **headers are extra pieces of information** sent with HTTP requests and responses.

👉 They help the browser and server understand how to handle data properly.

---

## Important Note

- Headers are not strictly required
- But without them, websites may not work correctly

---

# Request Headers (Client → Server)

These are sent from your browser to the server.

---

## Host
- Specifies which website you want to access
- Important when multiple websites share one server

---

## User-Agent
- Identifies your browser and version
- Helps server display website correctly

Example:
- Chrome, Firefox, etc.

---

## Content-Length
- Shows how much data is being sent
- Helps server know when request ends

---

## Accept-Encoding
- Tells server what compression formats your browser supports
- Helps reduce data size for faster loading

---

## Cookie
- Stores small pieces of data on your device
- Sent back to server on each request
- Used for login sessions and preferences

---

# Response Headers (Server → Client)

These are sent from server back to browser.

---

## Set-Cookie
- Stores data in the browser
- Sent back to server in future requests

---

## Cache-Control
- Controls how long browser stores content
- Improves performance by avoiding repeated downloads

---

## Content-Type
- Tells browser what type of data is being sent
- Examples:
  - HTML
  - CSS
  - Images
  - JSON

👉 Helps browser display data correctly

---

## Content-Encoding
- Shows how data is compressed
- Browser uses this to decompress content

---

# Simple Flow

```
Browser sends Request Headers → Server processes → Server sends Response Headers
```

---

# Key Takeaway

HTTP headers are **extra instructions** that help browsers and servers communicate efficiently, handle data correctly, and improve performance.
