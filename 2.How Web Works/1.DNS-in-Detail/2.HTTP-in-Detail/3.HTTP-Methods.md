# HTTP Methods

## Overview

HTTP methods tell a web server **what action the client wants to perform** when making a request.

---

## 1. GET

### Purpose:
Used to **retrieve (get) data** from a server.

### Key Idea:
- Only reads data
- Does NOT change anything on the server

### Example:
- Loading a webpage
- Fetching images or files

---

## 2. POST

### Purpose:
Used to **send data to a server**.

### Key Idea:
- Creates new data or submits information
- Common in forms (login, signup)

### Example:
- Creating an account
- Submitting a login form

---

## 3. PUT

### Purpose:
Used to **update existing data** on a server.

### Key Idea:
- Replaces or modifies existing information

### Example:
- Updating a user profile
- Editing a blog post

---

## 4. DELETE

### Purpose:
Used to **remove data from a server**.

### Key Idea:
- Permanently deletes a record

### Example:
- Deleting a user account
- Removing a post

---

## Simple Comparison Table

| Method | Action |
|--------|--------|
| GET | Retrieve data |
| POST | Create new data |
| PUT | Update existing data |
| DELETE | Remove data |

---

## Key Takeaway

HTTP methods define **what action you want the server to perform**, such as reading, creating, updating, or deleting data.
