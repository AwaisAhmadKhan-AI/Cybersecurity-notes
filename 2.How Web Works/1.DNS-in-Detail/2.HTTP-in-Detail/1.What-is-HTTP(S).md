# What is HTTP/HTTPS?

## HTTP (HyperText Transfer Protocol)

### What it is:
**HTTP** is a set of rules used to transfer data between a web browser and a web server.

---

### What it does:
- Loads websites in your browser
- Transfers web content like:
  - HTML (web pages)
  - Images
  - Videos
  - Files

---

### Simple Idea:
When you open a website:
- Your browser sends an HTTP request
- The server responds with website data

---

### Key Point:
- HTTP is **not secure**
- Data is sent in plain text (can be intercepted)

---

## HTTPS (HyperText Transfer Protocol Secure)

### What it is:
**HTTPS** is the secure version of HTTP.

---

### What it does:
- Same as HTTP but with **encryption**
- Protects data while it travels over the internet

---

### Why HTTPS is Important:

#### 1. Encryption
- Data is scrambled
- Hackers cannot read it easily

---

#### 2. Authentication
- Confirms you are talking to the **real website**
- Prevents fake/impersonation websites

---

#### 3. Data Integrity
- Ensures data is not modified during transfer

---

## HTTP vs HTTPS

| HTTP | HTTPS |
|------|-------|
| Not secure | Secure |
| No encryption | Encrypted |
| Data can be intercepted | Protected data |
| Uses port 80 | Uses port 443 |

---

## Simple Analogy

- HTTP = Sending a postcard (anyone can read it)
- HTTPS = Sending a locked box (only receiver can open it)

---

## Key Takeaway

**HTTP is used to load websites, while HTTPS is the secure version that encrypts data and protects communication between browser and server.**
