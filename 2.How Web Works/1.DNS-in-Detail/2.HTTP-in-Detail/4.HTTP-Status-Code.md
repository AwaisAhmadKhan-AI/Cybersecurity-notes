# HTTP Status Codes

## Overview

When a server responds to a request, it includes a **status code** in the first line of the response.

👉 This code tells the client whether the request was successful or not.

---

# Status Code Categories

## 1xx — Information Response
- Request received, continue processing
- Rare in modern web usage

---

## 2xx — Success
- Request was successful

---

## 3xx — Redirection
- Client is redirected to another resource or location

---

## 4xx — Client Errors
- Problem with the request sent by the client

---

## 5xx — Server Errors
- Problem on the server side

---

# Common HTTP Status Codes

## 200 — OK
- Request succeeded
- Everything worked correctly

---

## 201 — Created
- New resource successfully created
- Example: new user account or post

---

## 301 — Moved Permanently
- Resource has permanently moved to a new URL
- Browser automatically redirects

---

## 302 — Found (Temporary Redirect)
- Resource temporarily moved
- May change again later

---

## 400 — Bad Request
- Request is incorrect or missing required data
- Server cannot process it

---

## 401 — Unauthorized
- User must log in first
- Authentication required

---

## 403 — Forbidden
- Access is denied even if logged in
- No permission to view resource

---

## 404 — Not Found
- Resource does not exist on server

---

## 405 — Method Not Allowed
- Wrong HTTP method used
- Example: GET used instead of POST

---

## 500 — Internal Server Error
- Server crashed or failed unexpectedly
- Problem on server side

---

## 503 — Service Unavailable
- Server is overloaded or under maintenance
- Temporary issue

---

# Key Takeaway

HTTP status codes are **server messages** that tell the client what happened with their request — whether it succeeded, failed, or needs further action.
