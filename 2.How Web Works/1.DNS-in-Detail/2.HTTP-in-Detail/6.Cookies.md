# Cookies

## What are Cookies?

Cookies are **small pieces of data stored in your browser** by websites.

They are created when a server sends a:
```
Set-Cookie
```
header in the HTTP response.

---

## How Cookies Work

1. Server sends a cookie → browser stores it  
2. Browser sends cookie back with every request  
3. Server uses it to recognize the user  

---

## Why Cookies are Needed

HTTP is **stateless**, meaning:
- The server does NOT remember previous requests

Cookies fix this by allowing the server to:
- Remember who you are
- Store session data
- Save preferences
- Track visits

---

## Simple Flow

```
Server → Set-Cookie → Browser stores it
Browser → sends Cookie → Server
Server → identifies user
```

---

## Common Uses of Cookies

### 1. Authentication
- Keeps users logged in
- Uses a **token (secure identifier)** instead of password

---

### 2. Personalization
- Language settings
- Theme preferences
- User settings

---

### 3. Tracking
- Records user activity
- Used for analytics and website behavior

---

## Important Note

- Cookies usually do NOT store passwords
- They store **tokens (secure random values)**

👉 These tokens represent your login session

---

## How to View Cookies

You can inspect cookies using browser tools:

### Steps:
1. Open Developer Tools (F12)
2. Go to **Network tab**
3. Load a website
4. Click any request
5. Open **Cookies tab**

---

## Key Takeaway

Cookies are used to **store small data in your browser and maintain user sessions**, allowing websites to remember you even though HTTP itself is stateless.
