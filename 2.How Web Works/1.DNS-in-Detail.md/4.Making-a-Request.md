# What Happens When You Make a DNS Request

## Overview

When you type a website into your browser, a **DNS lookup process** starts to find the correct IP address.

This process involves multiple servers working together step by step.

---

## Step 1: Local Cache Check

- Your computer first checks its **local cache**
- If the domain was recently visited → IP is already saved
- If found → request ends here

👉 This makes browsing faster

---

## Step 2: Recursive DNS Server

If not found locally:
- Request is sent to a **Recursive DNS Server**
- Usually provided by your ISP

This server also checks its own cache:
- If found → returns result immediately
- If not found → continues the search

---

## Step 3: Root DNS Server

If recursive server has no answer:
- Request goes to the **Root DNS Server**

### Role:
- Directs request to correct **Top-Level Domain (TLD) server**

Example:
```
www.tryhackme.com → .com TLD
```

---

## Step 4: TLD Server

- Handles domains like `.com`, `.org`, `.net`, etc.
- Finds the correct **authoritative server**

👉 It does NOT give final IP, only direction

---

## Step 5: Authoritative DNS Server

- Also called **nameserver**
- Stores actual DNS records for the domain
- Example:
  - `tryhackme.com → IP address`

This is the FINAL source of truth.

---

## Step 6: Response Back to You

Flow:
1. Authoritative server sends IP to recursive server
2. Recursive server stores (caches) result
3. Response is sent back to your computer
4. Browser connects to the website

---

## Caching & TTL (Time To Live)

- DNS responses are stored temporarily in cache
- Controlled by **TTL value (in seconds)**

### Why TTL is important:
- Reduces repeated DNS lookups
- Speeds up future requests
- Reduces network load

---

## Simple Flow Summary

```
Browser → Local Cache
         ↓
Recursive DNS Server
         ↓
Root DNS Server
         ↓
TLD Server (.com, .org)
         ↓
Authoritative DNS Server
         ↓
IP Address Returned
```

---

## Key Takeaway

A DNS request is a **multi-step lookup process** that starts from your device and moves through several DNS servers until it finds the correct IP address, which is then cached for faster future access.
