# ASCII (American Standard Code for Information Interchange)

## Overview

Computers only understand:

```text
0 and 1
```

To store and display text, computers need a standard that assigns numbers to letters, digits, and symbols.

Example:

```text
A = 01000001
```

Whenever a computer reads:

```text
01000001
```

it knows the character is:

```text
A
```

This standard is called **ASCII**.

---

# What is ASCII?

**ASCII (American Standard Code for Information Interchange)** is a character encoding standard created in **1963**.

It assigns numbers to:

- Uppercase letters (A-Z)
- Lowercase letters (a-z)
- Digits (0-9)
- Punctuation symbols
- Control characters

ASCII uses:

```text
7 bits
```

This allows:

```text
2⁷ = 128
```

possible characters.

Range:

```text
0 - 127
```

---

# Why ASCII is Important

ASCII acts like a dictionary between:

```text
Text ↔ Numbers
```

Example:

```text
A ↔ 65
A ↔ 41 (Hex)
A ↔ 01000001 (Binary)
```

This allows all computers to store and display text consistently.

---

# Hexadecimal Representation

Binary is difficult for humans to read.

Therefore we convert every 4 bits into a hexadecimal digit.

Result:

```text
54 72 79 48 61 63 6B 4D 65 0A
```

This is how the file content often appears in:

- Hex editors
- Memory dumps
- Network packets
- Malware analysis tools

---

# Decimal Representation

The same text in decimal form:

```text
84 114 121 72 97 99 107 77 101 10
```

Example:

```text
T = 84
r = 114
y = 121
```

Although possible, decimal is rarely used in cybersecurity.

Hexadecimal is preferred.

# Why Cybersecurity Professionals Prefer Hexadecimal

Instead of:

```text
010101000111001001111001
```

We can write:

```text
54 72 79
```

Benefits:

- Easier to read
- Easier to analyze
- Shorter representation
- Common in networking and malware analysis

# Limitation of ASCII

ASCII only supports:

```text
English characters
```

Examples:

✔ A-Z

✔ a-z

✔ 0-9

✔ Symbols

But it cannot properly represent many international characters.

Examples:

```text
ñ
ü
ß
ł
č
ș
ț
```

---

# Extended Character Encodings

To support European languages, additional standards were created.

---

## ISO-8859-1 (Latin-1)

Supports Western European languages:

- German (ß, ü)
- French (é, ç)
- Spanish (ñ, ¿)
- Portuguese
- Italian
- Icelandic

---

## ISO-8859-2 (Latin-2)

Supports Central and Eastern European languages:

- Polish (ł, ń)
- Czech (č, ř)
- Hungarian (ő, ű)
- Romanian (ș, ț)
- Slovak
- Croatian

---

# Encoding Problem

If a file is saved using one encoding and opened using another:

```text
Saved as ISO-8859-1
Opened as ISO-8859-2
```

Special characters may appear incorrectly.

Example:

```text
ñ → strange symbol
```

This is called an:

```text
Encoding Mismatch
```

---

# ASCII vs Extended Encodings

| Feature | ASCII | ISO-8859 |
|-----------|--------|----------|
| Bits Used | 7 | 8 |
| Characters | 128 | 256 |
| English Support | Yes | Yes |
| European Languages | Limited | Better Support |

# Key Takeaways

- ASCII stands for **American Standard Code for Information Interchange**.
- ASCII uses **7 bits** and supports **128 characters**.
- Every character is represented by a unique numeric value.
- Computers store text as numbers, not actual letters.
- "TryHackMe" can be represented in Binary, Hexadecimal, or Decimal.
- Hexadecimal is the preferred format for viewing raw data.
- ASCII mainly supports English characters.
- Standards like **ISO-8859-1** and **ISO-8859-2** were created to support European languages.
- Understanding ASCII and Hexadecimal is essential for:
  - Digital Forensics
  - Packet Analysis
  - Malware Analysis
  - Reverse Engineering
  - Memory Analysis
  - Cybersecurity Investigations
