# Unicode

## Why Unicode Was Created

ASCII was limited to **7 bits (128 characters)** and mainly supported English letters, digits, and punctuation.

Extended ASCII attempted to solve this problem by adding an eighth bit, creating different regional standards such as:

- ISO-8859-1 (Latin-1)
- ISO-8859-2 (Latin-2)
- Windows-1252

However, this created compatibility problems.

### Example

A character saved as:

```text
Ø
```

using **ISO-8859-1 (Latin-1)** could appear as:

```text
Ř
```

when opened using **ISO-8859-2 (Latin-2)**.

This happened because different encoding standards assigned different meanings to the same byte values.

---

## The Need for a Universal Standard

Many languages require far more characters than English.

### Examples

- English: 26 letters (52 including uppercase and lowercase)
- Arabic: More than 250 characters including ligatures and diacritics
- Japanese:
  - 2,136 daily-use Kanji
  - JIS X 0208 defines 6,879 characters
- Chinese:
  - Educated speakers recognize around 8,000 characters
  - GB 18030-2022 defines more than 87,887 Hanzi characters
- Emoji: Thousands of symbols used in modern communication

A universal character encoding system became necessary.

---

# What is Unicode?

**Unicode** is a universal character encoding standard that assigns a unique code point to every character from every language and writing system.

### Benefits

- Supports all major world languages
- Supports historical writing systems
- Supports symbols and emojis
- Allows multiple languages in the same document
- Eliminates encoding compatibility issues

---

## Unicode Code Points

Every character is assigned a unique identifier called a **code point**.

### Examples

| Character | Unicode |
|------------|----------|
| A | U+0041 |
| Ω | U+03A9 |
| あ | U+3042 |

### Format

```text
U+XXXX
```

Where:

- U+ indicates Unicode
- XXXX represents the hexadecimal code point

# Unicode Encoding Formats

Unicode defines characters, but computers still need a method to store them in memory.

Common encoding formats:

1. UTF-8
2. UTF-16
3. UTF-32

---

# UTF-8

## Overview

UTF-8 is the most widely used Unicode encoding format, especially on the modern web.

### Characteristics

- Uses 1 to 4 bytes per character
- Variable-length encoding
- Backward compatible with ASCII

### Examples

| Character | Unicode | Bytes Used |
|------------|----------|------------|
| A | U+0041 | 1 Byte |
| Ω | U+03A9 | 2 Bytes |
| 🔥 | U+1F525 | 4 Bytes |

### Advantages

- Storage efficient
- Compatible with ASCII
- Internet standard

---

# UTF-16

## Overview

UTF-16 uses:

- 2 bytes for common characters
- 4 bytes for less common characters and emojis

### Examples

| Character | Encoding |
|------------|-----------|
| A | U+0041 |
| 🔥 | U+D83D U+DD25 |

### Advantages

- Efficient for many international languages
- Widely used in operating systems and applications

---

# UTF-32

## Overview

UTF-32 always uses exactly 4 bytes per character.

### Examples

| Character | Encoding |
|------------|-----------|
| A | U+00000041 |
| 🔥 | U+0001F525 |

### Advantages

- Very simple implementation
- Every character has the same length

### Disadvantages

- High storage consumption
- Less efficient than UTF-8 and UTF-16

# Key Terminology

## Unicode

A universal character encoding standard supporting all languages, symbols, and emojis.

## Code Point

A unique identifier assigned to every Unicode character.

Example:

```text
A → U+0041
```

## UTF-8

Variable-length Unicode encoding using 1–4 bytes.

## UTF-16

Unicode encoding using 2 or 4 bytes.

## UTF-32

Unicode encoding using 4 bytes for every character.
