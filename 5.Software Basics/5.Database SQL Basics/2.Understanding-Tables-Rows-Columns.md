# Understanding Tables, Rows, and Columns

## Overview

As businesses grow, they generate large amounts of information (data). Storing and managing this data using paper notebooks quickly becomes inefficient.

Databases solve this problem by organizing information into **tables**, which consist of **rows** and **columns**.

Understanding these three concepts is essential before learning SQL queries.

---

## The Café Example

Imagine a small café.

Initially, the café owner records every order in a paper notebook.

Each order contains:

* Drink name
* Price
* Time of order

Example:

| Drink  | Price | Time  |
| ------ | ----- | ----- |
| Coffee | 5     | 09:15 |
| Tea    | 3     | 09:30 |
| Latte  | 6     | 10:00 |

When the café is small, this works well.

However, after hundreds or thousands of orders, answering questions becomes difficult:

* How many coffees were sold today?
* What was the cheapest drink?
* How much money was earned this morning?

Searching manually through pages takes time.

This is why businesses use databases.

---

# What Is a Database?

A **database** is an organized collection of information stored electronically.

Think of it as a digital notebook that:

* Never runs out of pages
* Stores large amounts of information
* Allows fast searching
* Makes sorting and filtering easy

### Traditional Notebook vs Database

| Notebook              | Database                     |
| --------------------- | ---------------------------- |
| Limited pages         | Stores large amounts of data |
| Manual searching      | Instant searching            |
| Difficult to organize | Easy to organize             |
| Slow calculations     | Fast calculations            |

Even if the café has thousands of orders, the database can retrieve information within seconds.

---

# Tables

Inside a database, information is stored in **tables**.

A table looks similar to a spreadsheet.

### Example Café Orders Table

| Order ID | Drink  | Price | Time  |
| -------- | ------ | ----- | ----- |
| 1        | Coffee | 5     | 09:15 |
| 2        | Tea    | 3     | 09:30 |
| 3        | Latte  | 6     | 10:00 |

The entire structure above is called a **table**.

---

# Columns

Columns run **vertically** in a table.

They describe the type of information stored.

### Example

| Order ID | Drink | Price | Time |
| -------- | ----- | ----- | ---- |

The columns are:

* Order ID
* Drink
* Price
* Time

Each column stores one specific type of information.

### Examples

| Column   | Stores        |
| -------- | ------------- |
| Order ID | Order numbers |
| Drink    | Drink names   |
| Price    | Prices        |
| Time     | Order times   |

### Important Rule

> One column stores one type of information.

For example:

* The **Price** column stores only prices.
* The **Drink** column stores only drink names.

---

# Rows

Rows run **horizontally** across a table.

Each row contains one complete record.

### Example

| Order ID | Drink  | Price | Time  |
| -------- | ------ | ----- | ----- |
| 1        | Coffee | 5     | 09:15 |

This entire line is one row.

It represents one complete café order.

---

## Another Example

| Order ID | Drink | Price | Time  |
| -------- | ----- | ----- | ----- |
| 2        | Tea   | 3     | 09:30 |

This is a second row.

It contains all information about another customer order.

---

## Important Rule

> One row stores all information about one record.

In our café:

* One row = One order

If the café receives 10 orders:

```text
10 Orders = 10 Rows
```

If another customer places an order:

```text
11 Orders = 11 Rows
```

The database simply adds a new row.

# Asking Questions with SQL

A database becomes powerful when combined with SQL.

**SQL (Structured Query Language)** allows us to ask questions about data.

Instead of manually searching through thousands of rows, SQL lets the computer do the work.

---

## Example Questions

The café owner might ask:

### Show all orders

```sql
SELECT * FROM orders;
```

---

### Show only coffee orders

```sql
SELECT * FROM orders
WHERE drink = 'Coffee';
```

---

### Show the cheapest drink

```sql
SELECT * FROM orders
ORDER BY price ASC
LIMIT 1;
```

---

# What Is a Query?

A **query** is a request sent to a database.

Examples:

* Show all orders
* Show all coffee orders
* Show drinks sold today
* Show the cheapest drink

These requests are called queries.

---

## Important Note

Queries generally do **not** change data.

A query can simply retrieve and display information.

Example:

```sql
SELECT * FROM orders;
```

This only shows data.

It does not:

* Add rows
* Delete rows
* Modify rows

It simply retrieves information from the table.

---

# Key Concepts

* A database stores information in an organized way.
* Information is stored inside tables.
* Columns represent categories of data.
* Rows represent complete records.
* One row contains all information about one item.
* SQL is used to ask questions about data.
* Queries retrieve information from databases.

---

# Summary

## Database

An organized collection of data stored electronically.

## Table

A structured collection of rows and columns.

## Column

A category of information.

Examples:

* Drink
* Price
* Time

## Row

A complete record.

Example:

```text
Coffee | 5 | 09:15
```

represents one row.

## SQL

Structured Query Language used to interact with databases.

## Query

A request sent to a database to retrieve information.

### Example Query

```sql
SELECT * FROM orders;
```

This query displays all records from the `orders` table.

Understanding tables, rows, and columns is the foundation of working with databases and writing SQL queries.
