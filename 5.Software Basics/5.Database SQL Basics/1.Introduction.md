# Introduction to Databases and SQL

## Overview

Imagine a busy café that receives hundreds of orders every day.

At first, the café might record orders in a notebook. As the business grows, finding old orders, calculating sales, or tracking popular drinks becomes difficult and time-consuming.

Databases solve this problem by storing information in a structured format that is easy to organize, search, update, and manage.

---

## Why Data Matters

### What is Data?

**Data** is any piece of information that can be stored and processed by a computer.

### Examples of Data

| Type    | Example    |
| ------- | ---------- |
| Name    | Awais Khan |
| Age     | 21         |
| Product | Coffee     |
| Price   | $5         |
| Date    | 2025-11-15 |

Businesses rely on data to:

* Track customers
* Manage products
* Record sales
* Generate reports
* Make informed decisions

Without proper organization, managing large amounts of data becomes difficult.

---

## What Happens When a Computer Turns Off?

When a program is running, data is stored in memory (RAM).

However, RAM is temporary storage.

When the computer shuts down:

```text
RAM Data → Lost
```

To keep information permanently, it must be stored on a storage device such as:

* Hard Disk Drive (HDD)
* Solid State Drive (SSD)
* Database Server

This is where databases become important.

---

## What is a Database?

A **database** is an organized collection of data stored electronically.

It allows users to:

* Store information
* Retrieve information quickly
* Update existing records
* Delete records when necessary
* Search large datasets efficiently

### Real-World Examples

| Application  | Stored Data                        |
| ------------ | ---------------------------------- |
| Bank         | Customer accounts and transactions |
| Hospital     | Patient records                    |
| School       | Student information                |
| Online Store | Products and orders                |
| Café         | Customer orders and sales          |

---

## Why Use Databases?

Without a database:

```text
Customer1.txt
Customer2.txt
Customer3.txt
Customer4.txt
```

Managing thousands of files becomes difficult.

With a database:

```text
Customers Table
```

All information is stored in one structured location.

### Benefits

* Faster searching
* Better organization
* Easier updates
* Improved security
* Handles large amounts of data

---

## What is SQL?

**SQL** stands for:

```text
Structured Query Language
```

SQL is the standard language used to communicate with databases.

Using SQL, we can:

* Retrieve data
* Insert new data
* Update existing data
* Delete data
* Create new tables

Think of SQL as the language you use to ask questions from a database.

---

## Example SQL Query

```sql
SELECT * FROM customers;
```

### Meaning

* `SELECT` → Retrieve data
* `*` → Select all columns
* `FROM customers` → Get data from the customers table

Result:

| ID | Name    |
| -- | ------- |
| 1  | Alice   |
| 2  | Bob     |
| 3  | Charlie |

# Database Structure

Databases organize information using **tables**.

---

## What is a Table?

A table stores related information.

Example:

### Customers Table

| ID | Name    | Age |
| -- | ------- | --- |
| 1  | Alice   | 25  |
| 2  | Bob     | 30  |
| 3  | Charlie | 22  |

This table contains information about customers.

---

## What is a Row?

A **row** represents a single record.

Example:

| ID | Name  | Age |
| -- | ----- | --- |
| 1  | Alice | 25  |

This entire line is one row.

It represents one customer.

---

## What is a Column?

A **column** represents a category of information.

Example:

| ID | Name | Age |
| -- | ---- | --- |

Columns are:

* ID
* Name
* Age

Each column stores a specific type of data.

## How SQL Works

Suppose we have the following table:

| ID | Drink  | Price |
| -- | ------ | ----- |
| 1  | Coffee | 5     |
| 2  | Tea    | 3     |
| 3  | Latte  | 6     |

If we want to view all drinks:

```sql
SELECT * FROM drinks;
```

Result:

| ID | Drink  | Price |
| -- | ------ | ----- |
| 1  | Coffee | 5     |
| 2  | Tea    | 3     |
| 3  | Latte  | 6     |

The database processes the SQL query and returns the requested information.
