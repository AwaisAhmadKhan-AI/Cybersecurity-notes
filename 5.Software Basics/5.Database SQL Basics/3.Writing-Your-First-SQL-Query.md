# Writing Your First SQL Query

## Overview

Now that we understand what databases, tables, rows, and columns are, it's time to start communicating with a database using SQL.

SQL (**Structured Query Language**) allows us to retrieve, filter, and sort information stored in database tables.

In this lesson, we will learn four fundamental SQL keywords:

* `SELECT`
* `FROM`
* `WHERE`
* `ORDER BY`

These form the foundation of most SQL queries.

---

# Database Tables

The café database contains two tables:

## Orders Table

| Column |
| ------ |
| id     |
| drink  |
| price  |
| time   |

This table stores customer orders.

---

## Menu Table

| Column |
| ------ |
| drink  |
| price  |

This table stores available menu items and their prices.

---

# SQL Query Structure

A basic SQL query follows this pattern:

```sql
SELECT column_name
FROM table_name;
```

### Meaning

| Keyword | Purpose                      |
| ------- | ---------------------------- |
| SELECT  | Choose what data to display  |
| FROM    | Choose which table to search |

---

# Step 1: View Everything in a Table

To display all columns and all rows, use:

```sql
SELECT * FROM Orders;
```

### Understanding the Query

```sql
SELECT
```

Tells SQL what information to retrieve.

```sql
*
```

Means all columns.

```sql
FROM Orders
```

Tells SQL to retrieve data from the Orders table.

---

## Result

| id | drink  | price | time  |
| -- | ------ | ----- | ----- |
| 1  | Coffee | 5     | 09:15 |
| 2  | Tea    | 3     | 09:20 |
| 3  | Latte  | 6     | 09:30 |

This displays every row and every column.

---

# Step 2: Select Specific Columns

Sometimes we only need certain information.

Instead of selecting all columns, we can choose specific ones.

### Query

```sql
SELECT drink, price FROM Orders;
```

---

## Understanding the Query

```sql
SELECT drink, price
```

Retrieve only:

* drink
* price

```sql
FROM Orders
```

Get the data from the Orders table.

---

## Result

| drink  | price |
| ------ | ----- |
| Coffee | 5     |
| Tea    | 3     |
| Latte  | 6     |

Notice that:

* `id` is hidden
* `time` is hidden

Only the selected columns appear.

---

# Step 3: Filter Results with WHERE

The `WHERE` clause filters rows.

Instead of displaying every row, SQL only displays rows that match a condition.

### Query

```sql
SELECT * FROM Orders
WHERE drink = 'Coffee';
```

---

## Understanding the Query

```sql
WHERE drink = 'Coffee'
```

Means:

```text
Show only rows where the drink is Coffee
```

---

## Result

| id | drink  | price | time  |
| -- | ------ | ----- | ----- |
| 1  | Coffee | 5     | 09:15 |
| 8  | Coffee | 5     | 10:20 |
| 15 | Coffee | 5     | 11:05 |

Only coffee orders are displayed.

---

## Viewing Available Drinks

If you are unsure what drinks exist:

```sql
SELECT * FROM Menu;
```

This displays all available menu items.

---

# Step 4: Sort Results with ORDER BY

The `ORDER BY` clause sorts data.

### Query

```sql
SELECT * FROM Orders
ORDER BY price;
```

---

## Understanding the Query

```sql
ORDER BY price
```

Sorts rows by the price column.

By default:

```text
Ascending Order (Lowest → Highest)
```

---

## Example Result

| drink  | price |
| ------ | ----- |
| Tea    | 3     |
| Coffee | 5     |
| Latte  | 6     |

---

# Descending Order

To reverse the order, use:

```sql
DESC
```

### Query

```sql
SELECT * FROM Orders
ORDER BY price DESC;
```

---

## Understanding DESC

```text
DESC = Descending
```

Sort order becomes:

```text
Highest → Lowest
```

---

## Example Result

| drink  | price |
| ------ | ----- |
| Latte  | 6     |
| Coffee | 5     |
| Tea    | 3     |

---

# Step 5: Combining WHERE and ORDER BY

Real-world SQL queries often combine multiple clauses.

### Query

```sql
SELECT * FROM Orders
WHERE drink = 'Coffee'
ORDER BY price DESC;
```

---

## What Happens?

### Step 1

Filter:

```sql
WHERE drink = 'Coffee'
```

Keep only coffee orders.

### Step 2

Sort:

```sql
ORDER BY price DESC
```

Sort coffee orders from highest price to lowest price.

---

## Result

| id | drink  | price | time  |
| -- | ------ | ----- | ----- |
| 25 | Coffee | 6     | 12:00 |
| 14 | Coffee | 5     | 10:30 |
| 8  | Coffee | 4     | 09:45 |

Only coffee orders appear, sorted by price.

---

# SQL Query Execution Order

When reading SQL, think of it in this order:

```sql
SELECT columns
FROM table
WHERE condition
ORDER BY column;
```

### Example

```sql
SELECT drink, price
FROM Orders
WHERE drink = 'Coffee'
ORDER BY price DESC;
```

Meaning:

1. Look in the Orders table.
2. Keep only Coffee rows.
3. Sort by price (highest first).
4. Display drink and price columns.
