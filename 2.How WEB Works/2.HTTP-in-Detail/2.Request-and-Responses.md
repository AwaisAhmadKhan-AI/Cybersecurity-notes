# HTTP Requests and Responses

## Overview

When you open a website, your browser communicates with a web server by sending a **request** and receiving a **response**.

---

# What is a URL?

A **URL (Uniform Resource Locator)** tells the browser **where and how to access a resource** on the internet.

---

## Parts of a URL

Example:
```
http://user:password@tryhackme.com:80/view-room?id=1#task3
```

### 1. Scheme
- Defines protocol used
- Examples: `HTTP`, `HTTPS`, `FTP`

---

### 2. User (Optional)
- Username and password for authentication
- Rarely used in modern websites

---

### 3. Host
- Domain name or IP address
- Example: `tryhackme.com`

---

### 4. Port
- Specifies service port
- Default:
  - HTTP → 80
  - HTTPS → 443

---

### 5. Path
- Location of resource on server
- Example:
```
/view-room
```

---

### 6. Query String
- Extra information sent to server
- Example:
```
?id=1
```

---

### 7. Fragment
- Points to a section inside a page
- Example:
```
#task3
```

---

# HTTP Request

## Simple Request Example

```
GET / HTTP/1.1
Host: tryhackme.com
User-Agent: Mozilla/5.0 Firefox/87.0
Referer: https://tryhackme.com/
```

---

## Breakdown

### Line 1
- `GET` → Request method (fetch data)
- `/` → Home page
- `HTTP/1.1` → Protocol version

---

### Line 2
- Specifies target website:
  ```
  tryhackme.com
  ```

---

### Line 3
- Browser info (User-Agent)
- Example: Firefox version

---

### Line 4
- Refers to previous page visited

---

### Line 5
- Blank line → ends request

---

# HTTP Response

## Example Response

```
HTTP/1.1 200 OK
Server: nginx/1.15.8
Date: Fri, 09 Apr 2021 13:34:03 GMT
Content-Type: text/html
Content-Length: 98
```

---

## Breakdown

### Line 1
- HTTP version
- Status code:
  - `200 OK` → success

---

### Line 2
- Web server type and version

---

### Line 3
- Server date and time

---

### Line 4
- Type of content returned:
  - HTML, images, videos, etc.

---

### Line 5
- Size of response data

---

### Line 6
- Blank line → marks end of headers

---

### Lines 7+
- Actual webpage content (HTML)

---

# Simple Flow

```
Browser → Sends HTTP Request → Server
Server → Sends HTTP Response → Browser
```

---

# Key Takeaway

- **Request = Browser asking for data**
- **Response = Server sending data back**
- URLs define *where and how* to access resources on the internet
