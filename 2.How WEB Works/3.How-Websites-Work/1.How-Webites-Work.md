# How the Web Works

## Overview

When you visit a website, your browser communicates with a **web server** to get the data needed to display the page.

---

## Basic Process

1. You enter a website (URL)
2. Your browser sends a request to a web server
3. The server processes the request
4. The server sends back a response
5. Your browser displays the website

---

## What is a Web Server?

A **web server** is a computer located somewhere on the internet that:
- Receives requests from browsers
- Sends back website data

---

## Two Main Parts of a Website

### 1. Front End (Client-Side)

- Runs in your browser
- What the user sees and interacts with
- Includes:
  - HTML (structure)
  - CSS (design)
  - JavaScript (functionality)

👉 Example: buttons, text, images, layout

---

### 2. Back End (Server-Side)

- Runs on the web server
- Handles logic and data processing
- Sends responses to the browser

👉 Example: login system, database, authentication

---

## Simple Flow

```
Browser → Request → Web Server
Web Server → Response → Browser
Browser → Displays Website
```

---

## Key Idea

- Browser = Client (you)
- Server = Provider (website system)

---

## Key Takeaway

The web works by your browser sending a request to a server, and the server responding with data that your browser uses to display a website.
