# HTML Injection

## What is HTML Injection?

HTML Injection happens when a website **accepts user input and directly displays it on a page without filtering it**.

---

## Root Cause

It occurs when:
- User input is NOT sanitized (cleaned/filtered)
- Input is directly inserted into the webpage
- Browser treats input as real HTML

---

## What is Input Sanitisation?

Input sanitisation means:
- Cleaning user input
- Removing or blocking harmful HTML/JavaScript
- Preventing code from being executed

---

## How the Vulnerability Works

1. User enters input in a form
2. Website takes input
3. Website displays it directly on page
4. Browser interprets it as HTML code

---

## Example Scenario

If a website asks:

> "What is your name?"

And user inputs:
```html
<h1>Hello</h1>
```

Instead of showing text, the browser renders:
# Hello

---

## Why This is Dangerous

Attackers can:
- Change page layout
- Inject fake content
- Trick users (phishing)
- Potentially execute JavaScript (in worse cases)

---

## Key Concept

👉 User input should NEVER be trusted

---

## Prevention

Developers should:
- Sanitize all user input
- Remove HTML tags if not needed
- Escape special characters
- Validate input before displaying it

---

## Simple Rule

If user input is displayed directly on a page → it can be exploited.

---

## Key Takeaway

HTML Injection occurs when user input is treated as real HTML instead of plain text, allowing attackers to control how the webpage looks or behaves.
