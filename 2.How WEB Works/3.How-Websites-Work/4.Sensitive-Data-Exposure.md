# Sensitive Data Exposure

## What is Sensitive Data Exposure?

Sensitive Data Exposure happens when a website **accidentally reveals private or important information** to users.

This usually occurs when data is:
- Left in **HTML source code**
- Visible in **JavaScript files**
- Not properly protected or removed

---

## Where It Can Be Found

You can often find exposed data in:
- Page source code (View Source)
- HTML comments
- JavaScript files
- Frontend hidden elements

---

## Example of Exposed Data

Developers might accidentally leave:
- Login credentials
- Hidden admin links
- API keys or tokens
- Internal system paths

---

## Why It is Dangerous

Attackers can use this information to:
- Log into restricted areas
- Access admin panels
- Move deeper into the system
- Exploit backend services

---

## Simple Example

```html
<!-- temp login: admin / password123 -->
```

Even though it's hidden in a comment, it is still visible in the source code.

---

## Security Risk

If sensitive data is exposed:
- It can lead to **unauthorized access**
- It may compromise the entire application

---

## Important Practice

When testing a web application:
- Always check **page source code**
- Inspect **JavaScript files**
- Look for hidden comments or links

---

## Key Takeaway

Sensitive Data Exposure occurs when private information is accidentally left visible in the frontend, making it easy for attackers to find and misuse it.
