# How Web Servers Work

## What is a Web Server?

A **web server** is software that:
- Listens for incoming requests
- Uses HTTP/HTTPS to respond
- Delivers web content to users

---

## Common Web Server Software

- Apache
- Nginx
- IIS
- NodeJS

---

## Root Directory

A web server serves files from a **root directory**:

### Common locations:
- Linux: `/var/www/html`
- Windows: `C:\inetpub\wwwroot`

---

### Example

If you request:
```
http://example.com/picture.jpg
```

The server returns:
```
/var/www/html/picture.jpg
```

---

## Virtual Hosts

### What they are:
Virtual hosts allow **one server to host multiple websites**.

---

### How it works:
- Server checks the requested domain (from HTTP headers)
- Matches it with configuration files
- Serves the correct website

---

### Example mapping:

| Domain | Folder |
|--------|--------|
| one.com | /var/www/website_one |
| two.com | /var/www/website_two |

---

## Static vs Dynamic Content

### Static Content
- Does NOT change
- Directly served by server

Examples:
- HTML files
- CSS
- Images
- JavaScript files

---

### Dynamic Content
- Changes based on user input or data
- Generated in real-time

Examples:
- Blogs
- Search results
- User dashboards

---

## Backend vs Frontend

### Frontend
- What user sees in browser
- HTML output

### Backend
- Runs on server
- Processes logic and data
- Generates frontend output

---

## Backend Languages

Examples:
- PHP
- Python
- Ruby
- NodeJS
- Perl

---

## What Backend Can Do

Backend languages can:
- Talk to databases
- Process user input
- Call external APIs
- Generate dynamic pages

---

## Example (PHP)

### URL:
```
http://example.com/index.php?name=adam
```

### Code:
```php
<html><body>Hello <?php echo $_GET["name"]; ?></body></html>
```

### Output:
```
Hello adam
```

---

## Key Idea

- User does NOT see backend code
- Only final output is shown in browser

---

## Key Takeaway

Web servers handle requests, serve files, and generate dynamic content using backend languages — making modern websites interactive and functional.
