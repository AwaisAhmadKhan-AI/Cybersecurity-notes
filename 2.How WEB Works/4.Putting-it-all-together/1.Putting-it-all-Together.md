# Putting It All Together

## Overview

When you load a website, several systems work together behind the scenes to make it happen.

---

## Step-by-Step Process

### 1. Finding the Server (DNS)
- Your browser needs the website’s **IP address**
- It uses **DNS (Domain Name System)** to translate a domain name into an IP address

---

### 2. Sending the Request (HTTP/HTTPS)
- Browser connects to the web server using:
  - **HTTP** or
  - **HTTPS (secure version)**
- It sends a request asking for the webpage

---

### 3. Server Processing
- The web server receives the request
- It processes it and prepares a response

---

### 4. Sending Back Data
The server responds with:
- HTML (structure)
- CSS (design)
- JavaScript (functionality)
- Images and other resources

---

### 5. Browser Rendering
- Your browser receives all data
- It combines everything
- Displays the final website

---

## Simple Flow

```
User → DNS → IP Address Found
User → HTTP Request → Web Server
Web Server → Response (HTML/CSS/JS)
Browser → Renders Website
```

---

## Key Idea

Multiple technologies work together:
- DNS → finds address
- HTTP/HTTPS → transfers data
- HTML/CSS/JS → builds the website

---

## Key Takeaway

A website is not just one thing — it is a **combination of DNS, HTTP, servers, and frontend code working together to deliver a complete webpage.**
